# Auto-Wiki: レイアウト・テンプレート変更

wikiのレイアウト構造（ヘッダー、サイドバー、フッター、ナビゲーション、レスポンシブ設計）を変更するskill。

## Usage

```
/auto-wiki-layout [指示]
```

指示の例:
- `サイドバーを左に移動して`
- `ヘッダーに検索バーを追加して`
- `フッターにタグクラウドを表示して`
- `サイドバーを削除してシンプルにして`
- `記事テンプレートにパンくずリストを追加して`
- `index.htmlにカード形式の記事一覧を追加して`
- `モバイルでサイドバーをアコーディオンにして`

## 対象ファイル

| ファイル | 責務 |
|---|---|
| `templates/article.html` | 記事ページのHTML構造 |
| `templates/wiki-index.html` | wiki個別トップページのHTML構造 |
| `templates/portal.html` | 全wikiポータルのHTML構造 |
| `assets/css/wiki.css` | レイアウト関連CSS |
| `assets/js/graph.js` | グラフコンテナのサイズ・振る舞い |
| `assets/js/search.js` | 検索UIの配置変更時 |
| 既存の `wikis/*/articles/*.html` | テンプレート構造変更時の一括更新 |

## 実行手順

ユーザー入力: $ARGUMENTS

### 1. 現状把握

1. `templates/article.html` を読み込む → 現在の記事ページ構造を把握
2. `templates/wiki-index.html` を読み込む → wiki個別トップページ構造を把握
3. `templates/portal.html` を読み込む → ポータル構造を把握
4. `assets/css/wiki.css` を読み込む → レイアウト関連CSSを把握
5. 既存記事があれば `wikis/*/articles/` の件数を確認

### 2. 変更影響分析

変更内容に応じて影響範囲を分析:

| 変更種別 | 影響ファイル | 既存記事更新 |
|---|---|---|
| テンプレートHTMLの構造変更 | template + CSS | 必要（全記事） |
| CSSのみの変更 | CSSのみ | 不要 |
| JSの変更 | JS + 必要ならCSS | 不要 |
| プレースホルダーの追加 | template + 各skill | 必要（全記事） |

**既存記事の更新が必要な場合は、ユーザーに影響範囲を明示して確認を取る。**

### 3. テンプレート変更

#### 記事テンプレート（`templates/article.html`）の変更可能コンポーネント:

```
┌─────────────────────────────┐
│ wiki-header                  │ ← ヘッダー（サイトタイトル + ナビ）
├─────────────────────────────┤
│ breadcrumb                   │ ← パンくずリスト
├───────────────┬─────────────┤
│               │             │
│  wiki-main    │ wiki-sidebar│ ← メイン + サイドバー
│               │  - TOC      │
│  - title      │  - 関連記事 │
│  - meta       │             │
│  - summary    │             │
│  - content    │             │
│               │             │
├───────────────┴─────────────┤
│ wiki-footer                  │ ← フッター（リンク先/被リンク）
└─────────────────────────────┘
```

#### トップページテンプレート（`templates/index.html`）の変更可能コンポーネント:

```
┌─────────────────────────────┐
│ wiki-header                  │ ← ヘッダー
├─────────────────────────────┤
│ index-header                 │ ← タイトル + 説明
├─────────────────────────────┤
│ stats-bar                    │ ← 統計（記事数・リンク数）
├─────────────────────────────┤
│ search-container             │ ← 検索バー
├─────────────────────────────┤
│ graph-container              │ ← D3グラフ
├─────────────────────────────┤
│ article-list                 │ ← 記事一覧テーブル
├─────────────────────────────┤
│ wiki-footer                  │ ← フッター
└─────────────────────────────┘
```

### 4. レイアウトパターン

以下のレイアウトパターンを提供:

#### サイドバー位置
- **右サイドバー**（デフォルト）: `.wiki-content { flex-direction: row; }`
- **左サイドバー**: `.wiki-content { flex-direction: row-reverse; }` + ボーダー調整
- **サイドバーなし**: サイドバーをメイン下部に統合、TOCはコンテンツ内に

#### ヘッダースタイル
- **シンプル**（デフォルト）: タイトル + ナビリンク
- **検索付き**: ヘッダーに検索バーを統合
- **固定ヘッダー**: `position: sticky; top: 0;`

#### 記事一覧形式（index.html）
- **テーブル形式**（デフォルト）: 表形式の一覧
- **カード形式**: グリッドレイアウトのカード
- **リスト形式**: コンパクトなリスト

#### フッター拡張
- **デフォルト**: リンク先/被リンクのみ
- **タグクラウド**: 関連トピックのタグ表示
- **ナビゲーション**: 前後の記事リンク

### 5. プレースホルダー管理

テンプレートに新しいプレースホルダーを追加する場合:

1. テンプレートに `{{NEW_PLACEHOLDER}}` を追加
2. `auto-wiki-create.md` の「テンプレート適用」セクションにプレースホルダーの説明を追加
3. `auto-wiki-expand.md` のサブエージェント指示にも追加
4. 既存の全記事を更新（新プレースホルダーにデフォルト値を設定）

**重要**: プレースホルダーを追加した場合、関連する他のskillファイルも更新すること。

### 6. 既存記事の一括更新

テンプレート構造が変わった場合:

1. `wikis/*/articles/` 内の全HTMLファイルを走査
2. 各記事から本文コンテンツ（`{{CONTENT}}`部分）とメタデータを抽出
3. 新しいテンプレートで再生成
4. `db/articles.json` のメタデータを保持
5. Agent toolで並列処理（記事数が多い場合）

抽出対象のデータ:
- `<h1>` からタイトル
- `.article-meta` から更新日時
- `.article-summary` からサマリー
- `.wiki-main` の `{{CONTENT}}` 部分（H2以降）から本文
- `.toc-list` からTOC
- `.footer-links` からリンク先/被リンク

### 7. レスポンシブ調整

レイアウト変更時は必ずレスポンシブ対応を確認:

- **768px以下**: サイドバー → コンテンツ下に配置
- **480px以下**: フォントサイズ・パディング調整
- グラフコンテナの高さ調整
- テーブルの横スクロール対応

### 8. 完了報告

- 変更したファイル一覧と変更内容
- 更新された既存記事数（テンプレート変更時）
- 新しいプレースホルダーの追加有無
- 他のskillファイルへの影響（プレースホルダー追加時）
- ブラウザで確認するよう案内
