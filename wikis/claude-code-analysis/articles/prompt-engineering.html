プロンプト設計

最終更新: 2026-03-31
Claude Code の各ツールは prompt.ts ファイルに詳細な使用指示を格納しており、AI の行動を精密に制御する。BashTool・FileEditTool・GrepTool・AgentTool・コーディネータモードなど各コンポーネントに固有のプロンプト設計パターンがある。セキュリティ・一貫性・効率性を重視した指示体系が構築されている。

プロンプト設計の概要

Claude Code では各ツールが prompt.ts ファイルに使用指示を格納している。これらのプロンプトは LLM が各ツールをいつ・どのように使用するかを制御する重要な設計要素である。プロンプト設計には一貫したパターンが見られ、セキュリティ・効率性・ユーザー体験の向上を目的としている。

BashTool プロンプト

BashTool のプロンプトは以下の重要な指示を含む。

  • find・grep・cat・head・tail・sed・awk・echo の代わりに専用ツールを使用する
  • 独立したコマンドは単一メッセージ内で複数ツール呼び出しとして実行する
  • 順次実行が必要な依存コマンドは && でチェーンする
  • 明示的に要求されない限り、破壊的 git コマンドを使用しない
  • 明示的に要求されない限り、コミットしない
  • main/master への force-push は絶対に行わない
  • サンドボックスモード: 一時ファイルに $TMPDIR を強制使用する

FileEditTool プロンプト

FileEditTool は以下の重要な制約を持つ。

  • 編集前に必ず FileReadTool で読み取ること（強制）
  • 読み取り出力の正確なインデント（タブ/スペース）を保持する
  • old_string はファイル内で一意でなければならない；ユニーク性を確保するため2〜4行の隣接する文脈を提供する
  • ファイル全体のリネームには replace_all を使用する
  • 絶対に必要でない限り新規ファイルを作成しない

GrepTool プロンプト

GrepTool は以下の指示を含む。

  • ripgrep をベースとして構築；常に bash の grep/rg の代わりに Grep を使用する
  • 完全な正規表現構文をサポート；クロスライン検索には multiline: true を使用する
  • 3つの出力モード: content・files_with_matches（デフォルト）・count
  • オープンエンドなマルチラウンド検索には Agent ツールを使用する

WebFetchTool プロンプト

WebFetchTool は著作権コンテンツへの配慮を持つ設計になっている。

  • 2段階コンテンツ取得: 事前承認ドメインはフルコンテンツ取得可能；その他は125文字以内の引用制限
  • 歌詞や著作権コンテンツの逐語的複製を禁止
  • HTML からマークダウンへの変換後に処理

AgentTool プロンプト

AgentTool は並行実行を強調する設計になっている。

  • 常にエージェント起動の3〜5語の説明を含める
  • エージェント結果をユーザーに表示する前に要約する
  • 並行実行: 「単一メッセージ内で複数のツール使用コンテントブロックを送信する」
  • フォーク機能: 「Don't peek（フォーク出力を覗き見しない）」「Never fabricate or predict fork results（フォーク結果を捏造・予測しない）」
  • 「スマートな同僚が部屋に入ってきたようにエージェントにブリーフィングする」

コーディネータシステムプロンプト

コーディネータモードは明確な並行化哲学を持つ。

  • 「並列性はあなたのスーパーパワー。ワーカーは非同期。可能な限り独立したワーカーを並行起動する」
  • 「すべてのプロンプトはワーカーが必要とするすべてを含む自己完結型でなければならない」
  • コンテキスト重複が高い場合はワーカーを再利用；コンテキストが分岐した場合は新規ワーカーを生成
  • タスク結果フォーマット: <task_id>・<status>・<summary>・<result> を含む XML ラップメッセージ

セッションメモリ更新プロンプト

セッションメモリの更新には厳格な構造が求められる。

  • 10の構造化セクションと厳格なヘッダー保持
  • セクションあたり約2000トークンの制限；重要度の低い詳細を削除して圧縮
  • 継続性のために「Current State」セクションを常に更新する
  • 並行 Edit ツール呼び出しを使用；編集後はプロセスを語らずに停止する

プロンプトキャッシュ安定性

Claude Code はプロンプトキャッシュを活用してコストを削減する。ツールの説明文は実行ごとに変化しないように設計されており、API のプロンプトキャッシュヒット率を最大化する。ツールセットのアセンブリ時には名前での重複排除を行い、順序の安定性を維持してキャッシュ効率を高める。
