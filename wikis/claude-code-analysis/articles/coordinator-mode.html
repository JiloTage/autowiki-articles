コーディネータモード

最終更新: 2026-03-31
コーディネータモード（src/coordinator/coordinatorMode.ts）はマルチエージェント並列処理を管理する Claude Code の高度な機能である。「並列性はスーパーパワー」という設計哲学のもと、独立したワーカーエージェントを並行起動して複雑なタスクを分散処理する。コーディネータ内部ツールのワーカーへのアクセス除外・自己完結型プロンプト要件・XML タスク結果フォーマットなどの設計原則を持つ。

コーディネータモード概要

コーディネータモードは src/coordinator/coordinatorMode.ts に実装された Claude Code のマルチエージェント処理機能である。複雑なタスクを独立した作業単位に分解し、複数のワーカーエージェントに並行して割り当てることで処理効率を大幅に向上させる。

設計哲学

コーディネータモードの中核的な設計哲学は「並列性はあなたのスーパーパワー」である。

「Parallelism is your superpower. Workers are async. Launch independent workers concurrently whenever possible.」

（並列性はあなたのスーパーパワー。ワーカーは非同期。可能な限り独立したワーカーを並行起動する。）

この哲学は Claude Code のプロンプト設計にも反映されており、独立したタスクは常に並行実行することが推奨される。

コーディネータ・ワーカーモデル

コーディネータモードは以下の役割分担で動作する。

コーディネータの役割

  • タスクの分析・分解
  • ワーカーエージェントへの作業割り当て
  • ワーカーの進捗モニタリング
  • 結果の統合・要約
  • タスクの依存関係管理

ワーカーの役割

  • 割り当てられた独立したタスクの実行
  • 自律的な判断・ツール使用
  • 結果の XML フォーマットでの返却

ツール制限

コーディネータの内部ツール（チーム管理・メッセージング）はワーカーへのアクセスから除外される。これによりワーカーがコーディネータの制御機能を誤用するリスクを排除する。

タスク通知フォーマット

タスクの完了通知は XML でラップされたユーザーメッセージとしてコーディネータに届く。

<task_id>{タスクID}</task_id>
<status>{completed|failed|timeout}</status>
<summary>{結果の要約}</summary>
<result>{詳細な結果}</result>

自己完結型プロンプト

コーディネータの重要な原則の一つは「すべてのプロンプトはワーカーが必要とするすべてを含む自己完結型でなければならない」という点である。ワーカーエージェントは親コンテキストにアクセスできないため、割り当て時のプロンプトにすべての必要な情報を含める必要がある。コーディネータは委任前に発見事項を自己完結型の仕様に統合する。

ワーカー再利用 vs 新規生成

コーディネータはワーカーを再利用するか新規生成するかを状況に応じて判断する。

  • 再利用: コンテキストの重複が高い場合（例: 同じファイルへの連続操作）
  • 新規生成: コンテキストが発散した場合（例: 全く異なるタスク領域）

この判断により、コンテキストキャッシュの効率と実行の独立性のバランスを取る。

AgentTool との関係

コーディネータモードはAgentTool を基盤として構築されている。AgentTool が個々のサブエージェントの生成・実行を担当するのに対し、コーディネータモードはより高レベルでの並列エージェント管理を提供する。コーディネータモードは AgentTool の上位抽象として機能する。
