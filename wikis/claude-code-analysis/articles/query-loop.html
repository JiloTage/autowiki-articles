クエリループ

最終更新: 2026-03-31
クエリループ（src/query.ts、68.7 KB）はマルチターンの LLM 処理を実装する非同期ジェネレータである。セットアップ・コンテキスト準備・モデル呼び出し・エラーリカバリー・ツール実行・継続の6フェーズで構成され、プロンプトが長すぎる場合や最大出力トークン数に達した場合のリカバリーメカニズムを備える。

クエリループ概要

クエリループは src/query.ts（68.7 KB）に実装された非同期ジェネレータで、Claude Code の LLM 通信の中核を担う。QueryEngine から呼び出され、ユーザーとの複数ターンの対話を処理する。

処理フェーズ

クエリループは以下の6フェーズで構成される。

フェーズ1: セットアップ

状態を初期化し、メモリ・スキルをプリフェッチし、モデルを設定する。GrowthBook フィーチャーフラグをチェックして条件付き機能を有効化する。

フェーズ2: コンテキスト準備

コンテキストに対して以下の変換を適用する。

  • snip/microcompact: 古いメッセージを削減・圧縮する
  • collapse: 類似メッセージをまとめる
  • autocompact: コンテキストが限界に近づいた場合に自動圧縮する

フェーズ3: モデル呼び出し

Anthropic API を呼び出してレスポンスをストリーミングする。フォールバックモデルのサポートにより、主要モデルが利用不可の場合に代替モデルで処理を継続できる。

フェーズ4: エラーリカバリー

回復可能なエラー（プロンプトが長すぎる・最大出力トークン数超過）を保留し、以下の戦略でリカバリーを試みる。

  • コンテキスト圧縮のドレイン
  • リアクティブ圧縮
  • トークンエスカレーション

フェーズ5: ツール実行

並行実行（ストリーミング）または順次実行でツールを呼び出す。ツールの isConcurrencySafe() によって実行方式が決定される。実行後に要約を生成する。

フェーズ6: 継続

次のターンをキューに入れるか、ターミナル状態を返して処理を終了する。

主要設計パターン

継続点での状態再構築

状態は継続点でリーズントラッキングとともに全体的に再構築される。これにより複雑な中断・再開シナリオを処理できる。

フィーチャーゲーテッドモジュール

条件付きインポートによるツリーシェイキング対応で、不要なコードを実行時に除外する。

using ステートメントによるリソース管理

すべてのジェネレータ終了パスで using ステートメントを使用してリソースのクリーンアップを保証する。

エラーハンドリング

クエリループは以下のエラータイプを特別に処理する。

  • prompt_too_long: コンテキスト圧縮を実行し再試行
  • max_output_tokens: トークンエスカレーションまたはモデル切り替え
  • APIエラー: src/services/api/withRetry.ts（28.2 KB）のバックオフリトライ
  • tool_use エラー: ツールエラーをメッセージとして LLM に返し継続

ストリーミング処理

クエリループはレスポンスをストリーミングで処理する。テキストチャンクは受信と同時に UI に表示され、ツール使用宣言も即座に処理が開始される。これによりユーザーは LLM の思考プロセスをリアルタイムで確認できる。
