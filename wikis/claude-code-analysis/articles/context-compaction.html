コンテキスト圧縮

最終更新: 2026-03-31
Claude Code のコンテキスト圧縮システム（src/services/compact/、11ファイル）はトークン制限への対応機構である。コアコンパクション（60 KB）・自動コンパクション・マイクロコンパクション・セッションメモリ対応コンパクション・圧縮プロンプトの5コンポーネントで構成される。クエリループのコンテキスト準備フェーズで適用され、長時間セッションでも安定した動作を保証する。

コンテキスト圧縮概要

コンテキスト圧縮は src/services/compact/ ディレクトリ（11ファイル）に実装されており、LLM のトークン制限に対応するためのシステムである。長時間の対話セッションでは会話履歴が膨大になり、コンテキストウィンドウを超えてしまう。コンテキスト圧縮はこの問題を解決するために会話履歴を要約・削減する。

圧縮戦略

クエリループのコンテキスト準備フェーズでは複数の圧縮戦略が適用される。

snip/microcompact

古いメッセージを削減・細粒度に圧縮する。microCompact.ts が細粒度の圧縮を担当し、重要度の低い詳細を省略しながら要点を保持する。

collapse

類似メッセージをまとめる。繰り返しパターンのあるメッセージ（例: 連続するツール呼び出しの結果）を統合して効率的に表現する。

autocompact

autoCompact.ts がコンテキストが制限に近づいた場合に自動的に圧縮を実行する。閾値に達すると compact.ts のフル圧縮を実行し、会話全体を要約する。

圧縮プロンプト

prompt.ts（16 KB）は圧縮に使用するプロンプトテンプレートを格納する。このプロンプトは LLM に対して会話履歴を要約する際の指示を提供し、重要な情報（タスク状態・重要な発見・エラー修正など）が保持されるよう誘導する。

セッションメモリとの統合

sessionMemoryCompact.ts はセッションメモリを考慮した圧縮を実装する。圧縮時にセッションメモリの10セクション構造を保護し、重要な構造化情報が圧縮によって失われないようにする。

エラー時の圧縮

クエリループはプロンプトが長すぎる（prompt_too_long）エラーが発生した場合に動的に圧縮を実行する。

  1. エラーを受信して保留する
  2. 圧縮ドレインを実行（段階的に圧縮量を増やす）
  3. リアクティブ圧縮を適用（全体を再圧縮）
  4. トークンエスカレーション（大きなコンテキストウィンドウのモデルに切り替え）

品質保持

圧縮の際には以下の情報が優先的に保持される。

  • タスクの仕様・要件
  • 現在の作業状態
  • 重要なファイル・コードの変更
  • エラーと修正のパターン
  • ユーザーの指示・制約

一方、詳細なツール実行ログや冗長な繰り返し情報は優先度が低く、圧縮で削減される。
