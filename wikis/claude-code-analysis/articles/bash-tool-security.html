BashTool セキュリティ設計

最終更新: 2026-03-31
Claude Code の BashTool セキュリティ設計は、シェルコマンドを AI が安全に実行するための多層防御アーキテクチャである。bashSecurity.ts・pathValidation.ts・sedValidation.ts の3つの専用バリデーションモジュールが協調し、危険なコマンド・パス操作・ファイル破壊を防ぐ。サンドボックスモード・タイムアウト制御・専用ツールへの誘導設計が組み合わさり、AI によるシェル実行リスクを最小化している。

ファイル構成

BashTool セキュリティ設計は src/tools/BashTool/ ディレクトリ内の複数ファイルで実装されている。各ファイルは単一責任の原則に従い、セキュリティの異なる側面を担当する。

ファイル役割概要
BashTool.tsコア実装シェルコマンド実行・タイムアウト制御・サンドボックスモード管理
bashSecurity.tsコマンドセキュリティ検証危険なコマンドパターンの検出・ブロック・警告生成
pathValidation.tsパスバリデーションパストラバーサル・危険なディレクトリへのアクセス防止
sedValidation.tssed コマンド検証sed による意図しないファイル破壊パターンの検出
prompt.tsAI 向け指示専用ツールへの誘導・使用禁止コマンドの明示
UI.tsxターミナル UI実行コマンド・出力のレンダリング

この分離設計により、セキュリティロジックが独立してテスト・メンテナンス可能な構造になっている。ツールシステム全体でも、ツールディレクトリ内のファイル分離は共通パターンである。

コマンドセキュリティ検証（bashSecurity.ts）

bashSecurity.ts は BashTool のコマンドセキュリティ検証の中核を担う。AI が生成したコマンド文字列を実行前に解析し、危険なパターンを検出する。

検証の仕組み

コマンドは実行前に静的解析にかけられる。パイプ・リダイレクト・コマンド置換などのシェル構文を考慮した上で、以下のカテゴリのパターンを検出する。

  • 破壊的ファイル操作 — rm -rf /・rm -rf ~ などのルートや重要ディレクトリの削除
  • 権限昇格 — sudo を用いた予期しない権限変更
  • ネットワーク外部通信 — 機密情報の外部送信を示すパターン
  • プロセス管理の悪用 — フォーク爆弾や無制限なバックグラウンドプロセス生成
  • 環境変数の悪用 — 重要な環境変数の意図しない上書き

検出されたパターンは即座にブロックされるか、ユーザーへの警告とともに確認を求める。検証は プロンプト設計と連携し、AI が危険なコマンドを生成しないよう事前に誘導する。

パスバリデーション（pathValidation.ts）

pathValidation.ts は危険なパス操作を防止する専用モジュールである。BashTool だけでなく、FileEditTool や FileWriteTool からも利用される共有バリデーションレイヤーを形成する。

主要な検証項目

脅威対策
パストラバーサル../ を含む相対パスの正規化・検証
システムディレクトリアクセス/etc・/sys・/proc などの書き込み禁止
ホームディレクトリ保護~/.ssh・~/.config などの機密ディレクトリへの制限
シンボリックリンク悪用シンボリックリンクの解決後パスを検証し、保護対象への迂回を防止
絶対パスの検証許可されたワーキングディレクトリ外のアクセス制限

パスバリデーションはエージェントアーキテクチャのパーミッションスコーピングと連携する。サブエージェントが親エージェントのスコープ外のファイルにアクセスすることを防ぐ役割も担う。

sed コマンド検証（sedValidation.ts）

sedValidation.ts は sed コマンドの安全でない使用パターンを専門的に検証するモジュールである。sed はテキスト処理に強力なツールだが、誤った使用でファイルを意図せず破壊するリスクを持つ。

検出する危険パターン

  • インプレース編集リスク — sed -i によるバックアップなしのファイル直接編集
  • 過度に広いマッチ — 全行を対象とする正規表現パターンによる意図しない一括置換
  • デリミタの混乱 — パス文字列を含む置換でデリミタが重複するパターン
  • 空置換 — コンテンツを削除してしまう可能性のある空文字列への置換

BashTool の プロンプト設計では、sed の代わりに FileEditTool の使用を優先するよう AI を誘導している。sedValidation.ts はその誘導をバイパスして sed が使用された場合の最後の防衛線として機能する。

専用ツールへの誘導設計

BashTool の設計上の重要な特徴として、汎用コマンドの使用を抑制し、より安全な専用ツールへ AI を誘導する仕組みがある。この誘導はプロンプトレベルとランタイムレベルの二段階で実装されている。

この誘導設計には以下の利点がある。

  • 安全性 — 専用ツールは入力バリデーション・出力サニタイズが組み込まれている
  • 可視性 — 専用ツール呼び出しは UI に明確に表示され、ユーザーが監視しやすい
  • パーミッション統合 — 専用ツールは checkPermissions() でアクセス制御が一元管理される
  • 読み取り専用フラグ — isReadOnly() が true のツールは破壊的操作をしないことが保証される

サンドボックスモードでの $TMPDIR 強制

サンドボックスモードが有効な場合、BashTool はファイル書き込みを $TMPDIR 配下に強制する。これは自動化・CI 環境や untrusted コードを実行する場面での安全性を確保するための機能である。

$TMPDIR 強制の仕組み

サンドボックスモード下では以下の制約が適用される。

  • ファイル書き込み操作は $TMPDIR（通常 /tmp）配下のみ許可
  • プロジェクトディレクトリ外へのファイル作成を pathValidation.ts と連携してブロック
  • 環境変数 $TMPDIR が未設定の場合はシステムデフォルトの一時ディレクトリを使用
  • セッション終了後に一時ファイルのクリーンアップを実施

サンドボックスモードの設定は エージェントアーキテクチャのパーミッションシステムと連動し、checkPermissions() メソッドを通じて適用される。

タイムアウト制御

BashTool はコマンド実行のタイムアウトを制御する仕組みを持つ。長時間実行されるコマンドや無限ループへの対策として設計されている。

タイムアウト設計

項目内容
デフォルトタイムアウトコマンドごとに設定された上限時間（秒単位）
ユーザー指定タイムアウト実行時オプションでタイムアウト値のカスタマイズが可能
タイムアウト時の処理プロセスツリー全体の安全な終了（SIGTERM → SIGKILL）
進捗コールバック長時間実行中は定期的に進捗を call() のコールバックで通知
出力バッファ制限大量の出力を生成するコマンドに対するバッファサイズ制限

タイムアウト制御は クエリループのツール実行フェーズと連携し、タイムアウトが発生した場合は適切なエラーメッセージをクエリループに返却する。これにより AI は状況を把握し、代替手段を試みることができる。

セキュリティ設計の総括

BashTool のセキュリティ設計は、単一の防衛線ではなく多層防御（Defense in Depth）アプローチを採用している。プロンプトレベルの誘導・静的コマンド検証・パスバリデーション・サンドボックス・タイムアウトの各層が独立して機能しつつ、相互に補完し合う設計となっている。

  • プロンプト層（prompt.ts） — AI が危険なコマンドを生成しないよう事前誘導
  • 検証層（bashSecurity.ts・pathValidation.ts・sedValidation.ts） — 生成されたコマンドの実行前検証
  • 実行制御層（BashTool.ts） — サンドボックス・タイムアウトによる実行時制御
  • パーミッション層（checkPermissions()） — ユーザー設定に基づくアクセス制御

この設計は ツールシステム全体の安全な AI 操作という設計哲学の具体的実装であり、BashTool が最もリスクの高いツールであるがゆえに特に厚いセキュリティ層が設けられている。
