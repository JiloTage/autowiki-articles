Claude Code 概要

最終更新: 2026-03-31
Claude Code は Anthropic が開発したAIコーディングアシスタントのCLIツールであり、TypeScriptで実装された約51万行のコードベースを持つ。エージェント型アーキテクチャを採用し、ファイル操作・シェル実行・Web検索など多様なツールをAIが自律的に活用できる設計になっている。

Claude Code とは

Claude Code は Anthropic が開発した AI コーディングアシスタントの CLI（コマンドラインインターフェース）ツールである。2026年3月31日に npm 配布パッケージの .map ファイルを通じてソースコードが公開され、JiloTage/claude-code リポジトリにアーカイブされた。

コードベースは TypeScript で実装されており、約1,900のファイルと51万2千行以上のコードから構成される。主なディレクトリとして src/ 配下に tools・services・commands・bridge・skills・coordinator など34のサブディレクトリが存在する。

主要コンポーネント

Claude Code のアーキテクチャは複数の層から構成されている。

エントリポイント層

src/entrypoints/cli.tsx（39.3 KB）が CLI のエントリポイントであり、コマンドライン引数の解析・初期化・アプリケーションのブートストラップを担当する。src/main.tsx（803.9 KB）がコアアプリケーションの初期化を行う最大のファイルである。

クエリエンジン層

QueryEngine（src/QueryEngine.ts、46.6 KB）が会話のライフサイクルを管理し、クエリループ（src/query.ts、68.7 KB）がマルチターンの LLM 処理を実装する。

ツール実行層

ツールシステムは29の常時利用可能なコアツールと、フィーチャーフラグで制御される条件付きツールで構成される。各ツールは独自のディレクトリ（例: src/tools/BashTool/）を持ち、実装・UI・プロンプト・ユーティリティが分離されている。

サービス層

Anthropic API クライアント・MCPプロトコル・セッションメモリ・コンテキスト圧縮・LSP・OAuth・アナリティクスなど多様なサービスが含まれる。

ブリッジ層

ブリッジシステム（src/bridge/、33ファイル）が Claude Code CLI と IDE 拡張機能（VS Code、JetBrains）間の双方向通信を実現する。

アーキテクチャ全体像

Claude Code のデータフローは以下のように構成されている：

CLI エントリ (entrypoints/cli.tsx)
    ↓
QueryEngine（セッション・コスト・パーミッション管理）
    ↓
クエリループ（マルチターンジェネレータ + 圧縮）
    ↓
ツール実行層
    ├── ファイルツール (Read/Edit/Write/Glob/Grep)
    ├── BashTool（サンドボックスシェル実行）
    ├── AgentTool（サブエージェント生成）
    ├── WebFetch/WebSearch（インターネットアクセス）
    ├── MCP ツール（プロトコルブリッジ外部ツール）
    └── Skill/Command ツール
    ↓
サービス層
    ├── Anthropic API (claude.ts - ストリーミング、リトライ)
    ├── MCP プロトコル（サーバー管理、認証、チャネル）
    ├── セッションメモリ（10セクション構造化メモリ）
    ├── 圧縮サービス（トークン管理）
    └── アナリティクス（テレメトリ、GrowthBook フィーチャーフラグ）
    ↓
ブリッジ層（IDE 拡張機能 ↔ CLI）

主要設計原則

  • エージェント型アーキテクチャ: AI がツールを自律的に選択・実行して複雑なタスクを完遂する
  • 並行実行: 独立したツール呼び出しを並行実行し、パフォーマンスを最大化する
  • パーミッション制御: 各ツールが checkPermissions() を実装し、アクセス制御を徹底する
  • コンテキスト管理: トークン制限に対応するため圧縮・要約機構を備える
  • フィーチャーフラグ: GrowthBook を用いた段階的なフィーチャーロールアウト
  • マルチエージェント: AgentTool によるサブエージェントの生成とコーディネータモードによる並列化
