QueryEngine

最終更新: 2026-03-31
QueryEngine（src/QueryEngine.ts、46.6 KB）は Claude Code の会話ライフサイクルを管理するコアコンポーネントである。可変メッセージ状態の維持・パーミッション拒否のトラッキング・API 使用量の管理・セッションのトランスクリプトストレージへの永続化を担当する。ask() ラッパーはステートレスな一発クエリを提供する。

QueryEngine 概要

QueryEngine は Claude Code における会話管理の中枢である。複数ターンにわたる対話において状態を保持し、LLM（大規模言語モデル）との通信を管理する。実装は src/QueryEngine.ts（46.6 KB）に格納されている。

主な責務

  • 可変メッセージ状態の維持: 複数ターンにわたる会話履歴を管理する
  • パーミッション拒否のトラッキング: ユーザーがツール実行を拒否した記録を保持する
  • API 使用量の管理: トークン消費量・コストを追跡する
  • Abort コントローラーの管理: キャンセル処理のためのコントローラーを管理する
  • セッション永続化: トランスクリプトをストレージに保存する

ask() ラッパー

QueryEngine は ask() メソッドを提供する。これはステートレスな一発クエリのための便利なラッパーであり、会話状態を持たない単純な質問・応答のユースケースに対応する。

クエリループとの関係

QueryEngine はクエリループ（src/query.ts）を内部的に使用する。クエリループが実際のマルチターン LLM 処理を担当する非同期ジェネレータであるのに対し、QueryEngine はそのライフサイクル全体を管理するラッパーレイヤーである。

  • QueryEngine → セッション・状態・パーミッションの管理
  • クエリループ → 実際の LLM 通信・ツール実行の制御

セッション永続化

QueryEngine はセッションをトランスクリプトストレージに永続化する。これにより Claude Code のセッションが中断された場合でも状態を復元できる。セッション再開時は保存されたトランスクリプトから状態を再構築する。

Abort コントローラー管理

QueryEngine は abort コントローラーを管理し、ユーザーやシステムからのキャンセルリクエストを処理する。エージェントモードでは、非同期サブエージェントが独立した abort コントローラーを持つことで、親エージェントのキャンセルがサブエージェントに伝播しないよう制御できる。

パーミッション拒否トラッキング

ユーザーがツール実行を拒否した場合、QueryEngine はその記録を保持する。これにより Claude は同じツール呼び出しを繰り返し試みることなく、代替アプローチを模索できる。拒否の記録はセッション内で維持され、会話の文脈として LLM に提供される。
