エージェントアーキテクチャ

最終更新: 2026-03-31
Claude Code のエージェントシステムは AgentTool を中心に構成されており、サブエージェントの生成・実行・管理を行う。ビルトイン・カスタム・プラグインの3カテゴリのエージェントをサポートし、MCP サーバー依存フィルタリング・パーミッションスコーピング・コンテキスト分離などの高度な機能を持つ。

AgentTool 概要

エージェントシステムの中核は src/tools/AgentTool/ ディレクトリに実装されており、以下のファイルで構成される。

エージェント実行フロー

runAgent.ts が定義するエージェントの実行フローは以下の通りである。

  1. 初期化: ユニークなエージェント ID を生成し、テレメトリを登録する
  2. 状態準備: ファイル状態キャッシュを準備し、パーミッションコンテキストを解決する
  3. 設定決定: パーミッションモード・ツールセット・MCP 接続を決定する
  4. フックとスキル: フロントマターフックを登録し、スキルを初期メッセージとしてプリロードする
  5. ストリーミング処理: クエリループからメッセージをストリームし、トランスクリプトをインクリメンタルに記録する
  6. クリーンアップ: MCP 接続を解放し、フックをクリアし、バックグラウンドタスクを終了し、エージェント状態を GC する

エージェントカテゴリ

Claude Code は3つのエージェントカテゴリをサポートする。

ビルトインエージェント

ハードコードされたエージェントで、動的なプロンプト生成コールバックを持つ。一般目的・プランニング・検証・探索の各用途に対応するエージェントが含まれる。

カスタムエージェント

ユーザー・プロジェクト・ポリシーが定義したエージェントで、プロンプトはクロージャに格納される。プロジェクト設定ファイルやユーザー設定から動的にロードされる。

プラグインエージェント

外部プラグインシステムから提供されるエージェント。プラグインシステムを通じてロードされ、通常のエージェントと同様に動作する。

ロード優先順位

loadAgentsDir.ts は以下の優先順位でエージェントをロードする（後のものが前のものを上書き）。

  1. ビルトイン
  2. プラグイン
  3. ユーザー
  4. プロジェクト
  5. フラグ
  6. ポリシー設定

主要設計パターン

コンテキスト分離

非同期エージェントはリンクが切れた abort コントローラーを取得し、同期エージェントは親コンテキストを共有する。これによりエージェントが独立して実行できる一方、必要に応じてキャンセルが可能になる。

パーミッションスコーピング

ツールセットはエージェントごとに制限されながら、SDK レベルのパーミッションは保持される。これにより細粒度のアクセス制御が実現される。

重複排除

ツールセットは名前でマージ・重複排除される。ビルトインツールが MCP ツールより優先される。

管理者信頼ゲート

フロントマター機能はロード前にソースの信頼性をチェックする。ユーザー定義設定はこのゲートを通過する必要がある。

MCP 依存フィルタリング

エージェントは必要な MCP サーバーが利用可能な場合のみロードされる。これにより設定ミスによるエラーを防止する。

サブエージェントフォーク

forkSubagent.ts は親エージェントの状態からサブエージェントを生成する機能を提供する。フォーク設計の原則は「Don't peek（フォーク出力を覗き見しない）」と「Never fabricate or predict fork results（フォーク結果を捏造・予測しない）」である。エージェントの出力は実際の実行結果のみを信頼する。

エージェントメモリ

agentMemory.ts と agentMemorySnapshot.ts がエージェントの実行中の状態を管理する。メモリはスナップショットとして保存され、エージェントの再開（resumeAgent.ts）時に復元される。
