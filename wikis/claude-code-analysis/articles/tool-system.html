ツールシステム

最終更新: 2026-03-31
Claude Code のツールシステムは Tool インターフェースを基盤とし、29の常時利用可能なコアツールと条件付きツールで構成される。各ツールは独立したディレクトリ構造を持ち、実装・UI・プロンプト・セキュリティ検証が分離されている。ツールは並行実行安全性・読み取り専用フラグ・パーミッションチェックを実装し、AI が安全にシステムを操作できる設計になっている。

Tool インターフェース

すべてのツールは src/Tool.ts（29.5 KB）で定義された Tool<Input, Output, P> インターフェースを実装する。このインターフェースは以下のメソッドを要求する。

  • call() — コンテキストと進捗コールバックを伴って実行する
  • description() — コンテキストに応じた説明文を生成する
  • checkPermissions() — アクセス制御を強制する
  • isConcurrencySafe() — 並行実行の安全性を宣言する
  • isReadOnly() — 破壊的操作かどうかをフラグで示す

ツールアセンブリ

src/tools.ts（17.3 KB）が getAllBaseTools() と assembleToolPool() を通じてツールを組み立てる。アセンブリプロセスでは名前での重複排除（ビルトインツールが MCP ツールより優先）とプロンプトキャッシュの安定性維持が行われる。

ツールディレクトリ構造

各ツールは独自のディレクトリに配置される（例: src/tools/BashTool/）。標準的な構成は以下の通り。

  • {ToolName}.ts または .tsx — コア実装
  • UI.tsx — ターミナル UI レンダリング
  • prompt.ts — ツールの説明・指示
  • utils.ts — ヘルパー関数
  • セキュリティ・バリデーションファイル（例: BashTool の bashSecurity.ts、pathValidation.ts、sedValidation.ts）

ツールディレクトリは計47個存在し、各ツールが独立した実装単位を形成している。

BashTool の設計

BashTool はセキュリティに特に注意が払われており、以下のファイルで構成される。

  • BashTool.ts — コアシェル実行
  • bashSecurity.ts — コマンドのセキュリティ検証
  • pathValidation.ts — パスのバリデーション
  • sedValidation.ts — sed コマンドの検証

BashTool は find、grep、cat、head、tail、sed、awk、echo など専用ツールで代替可能なコマンドの使用を抑制し、より安全な専用ツールへの誘導を行う。サンドボックスモードでは一時ファイルに $TMPDIR を強制使用する。

ファイルツールの設計

FileReadTool（39.1 KB）は通常のテキストファイルに加え、画像・PDF・Jupyter Notebook をサポートする。FileEditTool は編集前に必ず FileReadTool での読み取りを要求し（強制）、old_string がファイル内で一意であることを検証する。

Web ツールの設計

WebFetchTool は2段階のコンテンツ取得ポリシーを持つ。事前承認済みドメインはフルコンテンツを取得できるが、その他のドメインは125文字以内の引用制限が適用される。また著作権コンテンツの逐語複製を禁止する。コンテンツは HTML からマークダウンに変換されてから処理される。

ツール実行の並行性

isConcurrencySafe() メソッドが true を返すツールは並行実行可能である。Claude Code はツール呼び出しを可能な限り並行実行してパフォーマンスを最大化するが、依存関係のあるツール呼び出しは順次実行する。
