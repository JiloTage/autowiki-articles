Skillsの更新通知と変更管理の自動化

最終更新: 2026-03-31
Claude Code Skillsリポジトリの更新を開発者に確実に届けるための通知・変更管理ガイド。GitHub Actionsによるアラート自動化、RSS/Atomフィード生成、自動更新スクリプト、変更影響範囲の可視化、通知フィルタリング設計を体系的に解説する。

概要

Skillsの更新通知と変更管理の自動化は、セントラルリポジトリアプローチをはじめとするSkills共有運用において、更新が開発者の手元に確実・迅速に届く仕組みを整えるための実践ガイドだ。Skillsはコードと同様に継続的に改善されるため、変更をサイレントに放置するとチーム内でバージョンの断片化が起きる。適切な通知フローを設計することで、全員が常に最新のSkillsを使える環境を維持できる。

GitHub Actionsによる通知自動化

Slackチャンネルへのプッシュ通知

Skillsリポジトリへのマージ（`main`へのpush）をトリガーとして、GitHub ActionsがSlackに通知を送る構成が最もシンプルで信頼性が高い。以下のワークフローファイルをSkillsリポジトリの `.github/workflows/notify-slack.yml` に配置する：

name: Notify Skills Update

on:
  push:
    branches: [main]
    paths:
      - 'commands/**'

jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 2

      - name: Get changed files
        id: changes
        run: |
          CHANGED=$(git diff --name-only HEAD~1 HEAD -- commands/ | tr '\n' ', ' | sed 's/,$//')
          echo "files=$CHANGED" >> $GITHUB_OUTPUT

      - name: Post to Slack
        uses: slackapi/slack-github-action@v1
        with:
          payload: |
            {
              "text": ":wrench: *Claude Code Skills が更新されました*",
              "blocks": [
                {
                  "type": "section",
                  "text": {
                    "type": "mrkdwn",
                    "text": ":wrench: *Claude Code Skills が更新されました*\n変更ファイル: `${{ steps.changes.outputs.files }}`\nコミット: ${{ github.sha }}\n\n`bash ~/.claude-skills/install.sh` を実行して最新版を適用してください。"
                  }
                }
              ]
            }
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_SKILLS_WEBHOOK }}

PRマージ時の変更サマリー生成

PRがマージされる際、変更されたSkillファイルを自動で列挙してPR本文にコメントするアクションも有効だ。レビュアーと受け取り側の両方が変更内容を把握しやすくなる：

- name: Comment changed skills on PR
  uses: actions/github-script@v7
  with:
    script: |
      const { data: files } = await github.rest.pulls.listFiles({
        owner: context.repo.owner,
        repo: context.repo.repo,
        pull_number: context.payload.pull_request.number,
      });
      const skillFiles = files
        .filter(f => f.filename.startsWith('commands/'))
        .map(f => `- \`${f.filename}\` (${f.status})`)
        .join('\n');
      if (skillFiles) {
        await github.rest.issues.createComment({
          owner: context.repo.owner,
          repo: context.repo.repo,
          issue_number: context.payload.pull_request.number,
          body: `## 変更されたSkills\n${skillFiles}\n\nマージ後、チームメンバーに \`install.sh\` 再実行を案内してください。`,
        });
      }

RSS/Atomフィード生成

フィード自動生成スクリプト

Slack通知を受け取らないメンバーや、RSSリーダーで変更を追跡したいメンバー向けに、Skillsリポジトリの更新をAtomフィードとして公開できる。GitHub Actionsでコミットログから自動生成する例：

#!/usr/bin/env python3
# generate_feed.py - git logからAtomフィードを生成

import subprocess, datetime, xml.etree.ElementTree as ET

REPO_URL = "https://github.com/myorg/claude-skills"
FEED_PATH = "feed.xml"

def get_commits(n=20):
    result = subprocess.run(
        ["git", "log", f"-{n}", "--format=%H|%s|%ai", "--", "commands/"],
        capture_output=True, text=True
    )
    entries = []
    for line in result.stdout.strip().splitlines():
        sha, subject, date = line.split("|", 2)
        entries.append({"sha": sha, "subject": subject, "date": date})
    return entries

def build_feed(commits):
    feed = ET.Element("feed", xmlns="http://www.w3.org/2005/Atom")
    ET.SubElement(feed, "title").text = "Claude Code Skills Updates"
    ET.SubElement(feed, "id").text = REPO_URL
    ET.SubElement(feed, "updated").text = commits[0]["date"] if commits else ""
    link = ET.SubElement(feed, "link")
    link.set("href", REPO_URL)

    for c in commits:
        entry = ET.SubElement(feed, "entry")
        ET.SubElement(entry, "id").text = f"{REPO_URL}/commit/{c['sha']}"
        ET.SubElement(entry, "title").text = c["subject"]
        ET.SubElement(entry, "updated").text = c["date"]
        content = ET.SubElement(entry, "content")
        content.set("type", "text")
        content.text = f"Commit: {c['sha'][:8]}\n{c['subject']}"
    return ET.tostring(feed, encoding="unicode", xml_declaration=True)

if __name__ == "__main__":
    commits = get_commits()
    with open(FEED_PATH, "w") as f:
        f.write(build_feed(commits))
    print(f"Generated {FEED_PATH} with {len(commits)} entries.")

GitHub Actionsで `push` のたびにこのスクリプトを実行し、生成した `feed.xml` をリポジトリルートにコミットするか、GitHub Pagesで公開することで常に最新フィードが参照できる。

自動更新スクリプト

cron jobによる定期更新

セットアップ自動化スクリプトと連携して、定期的に最新Skillsへ自動更新する仕組みを構築できる。各開発者のマシンに以下の設定を加えるか、オンボーディングスクリプトに含める：

# ~/.config/cron/skills-update.sh
#!/bin/bash
set -euo pipefail

SKILLS_DIR="$HOME/.claude-skills"
LOG="$HOME/.claude-skills-update.log"

echo "[$(date -Iseconds)] Checking for updates..." >> "$LOG"

if [ ! -d "$SKILLS_DIR" ]; then
  echo "[$(date -Iseconds)] Skills dir not found, skipping." >> "$LOG"
  exit 0
fi

BEFORE=$(git -C "$SKILLS_DIR" rev-parse HEAD)
git -C "$SKILLS_DIR" pull --ff-only --quiet 2>> "$LOG"
AFTER=$(git -C "$SKILLS_DIR" rev-parse HEAD)

if [ "$BEFORE" != "$AFTER" ]; then
  echo "[$(date -Iseconds)] Updated: $BEFORE -> $AFTER" >> "$LOG"
  bash "$SKILLS_DIR/install.sh" >> "$LOG" 2>&1
  # macOS向け: terminal-notifier がインストール済みの場合
  if command -v terminal-notifier &>/dev/null; then
    terminal-notifier -title "Claude Skills 更新" \
      -message "新しいSkillsが適用されました" -sound default
  fi
else
  echo "[$(date -Iseconds)] Already up-to-date." >> "$LOG"
fi

crontabへの登録例（毎週月曜 9:00 に実行）：

# crontab -e
0 9 * * 1 bash ~/.config/cron/skills-update.sh

git hookを利用した更新トリガー

開発者がプロジェクトリポジトリで `git pull` を実行したタイミングに合わせて Skills を更新する方法もある。グローバルな `post-merge` フックを設定することで、git操作のついでに Skills が最新化される：

# ~/.config/git/hooks/post-merge
#!/bin/bash
# プロジェクトのpullに便乗してSkillsも更新
if [ -d "$HOME/.claude-skills" ]; then
  git -C "$HOME/.claude-skills" pull --ff-only --quiet 2>/dev/null || true
fi
# ~/.gitconfig に追加
[core]
  hooksPath = ~/.config/git/hooks

変更影響範囲の可視化

構造化CHANGELOGフォーマット

Skillsの変更が「誰に」「どのくらい影響するか」を伝えるため、CHANGELOG.mdを構造化して記載する。影響範囲タグを付けることで、開発者が自分に関係する変更をすぐに識別できる：

## [2026-03-31] - Skills v1.4.0

### 破壊的変更 🔴 [要再実行]
- `commands/deploy.md`: `--env` フラグの挙動を変更。`staging` → `stg` に短縮

### 機能追加 🟢
- `commands/review-pr.md`: レビューコメントの自動分類機能を追加
- `commands/generate-tests.md`: カバレッジレポート出力オプションを追加

### バグ修正 🟡
- `commands/run-migrations.md`: ロールバック時のエラーハンドリングを修正

### 影響対象チーム
| Skill | 影響チーム |
|---|---|
| deploy | Backend, DevOps |
| review-pr | 全チーム |
| generate-tests | Frontend, Backend |

差分サマリーの自動生成

GitHub Actionsでマージ時にコミット差分を解析し、影響するSkillファイルの一覧と変更規模（追加/削除行数）をREADMEの「最近の変更」セクションに自動更新することもできる：

- name: Update README recent changes
  run: |
    CHANGES=$(git diff HEAD~1 HEAD --stat -- commands/ \
      | grep '\.md' \
      | awk '{print "- " $1 " (" $2 " changes)"}')
    DATE=$(date +%Y-%m-%d)
    # README.md の ## 最近の変更 セクションを更新
    python3 scripts/update_readme_changes.py "$DATE" "$CHANGES"

通知フィルタリング設計

チーム別フィルタリング

全開発者が全Skillの更新通知を受け取ると通知疲れが起きる。Skillファイルにフロントマター形式でメタデータを記述し、通知先チームを絞り込む設計が効果的だ：

---
name: deploy
description: アプリケーションのデプロイを実行する
teams: [backend, devops]
breaking_change: false
---

# /deploy

...（Skill本文）

GitHub Actionsで変更されたファイルのメタデータを読み取り、対象チームのSlackチャンネルのみに通知を送ることができる：

changed_files.forEach(file => {
  const meta = parseSkillFrontmatter(file);
  meta.teams.forEach(team => {
    const webhook = TEAM_WEBHOOKS[team];
    if (webhook) postToSlack(webhook, file, meta);
  });
});

破壊的変更の優先アラート

破壊的変更（引数の変更、削除、動作の変更）は特別扱いして、通常のチャンネルに加えてDMや @here メンションで通知する。`breaking_change: true` のフラグが立っているSkillが変更された場合のみ強い通知を出すフィルター：

- name: Alert on breaking changes
  if: contains(steps.meta.outputs.flags, 'breaking_change=true')
  uses: slackapi/slack-github-action@v1
  with:
    payload: |
      {
        "text": ":rotating_light:  *破壊的変更があります* :rotating_light:\n`${{ steps.changes.outputs.files }}` が変更されました。\n*即時適用が必要です:* `bash install.sh`"
      }
  env:
    SLACK_WEBHOOK_URL: ${{ secrets.SLACK_SKILLS_URGENT_WEBHOOK }}

通知設計はSkills共有方法の概要で紹介した各アプローチ（Gitサブモジュールアプローチ、npm / GitHub Packages）にも適用できる汎用パターンであり、配布方法にかかわらずチーム全体のSkills活用度を高める基盤となる。
