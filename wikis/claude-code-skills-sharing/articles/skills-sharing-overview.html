Claude Code Skills共有方法 概要

最終更新: 2026-03-30
Claude Code用のSkillsをプライベートリポジトリで社内に広く共有するための方法を体系的に解説する。npm/GitHubパッケージ、Gitサブモジュール、モノレポ構成など複数のアプローチを比較検討する。

Claude Code Skillsとは

Claude Code Skills（スキルズ）とは、Claude Codeで利用可能なカスタムコマンドの拡張機能であり、`.claude/commands/` ディレクトリ内にMarkdownファイルとして定義される。各Skillは自然言語指示と許可ツールリストから構成され、`/skill-name` という形式でユーザーが呼び出すことができる。

Skillsは個人のローカル設定（`~/.claude/commands/`）またはプロジェクト固有設定（`.claude/commands/`）として配置できる。社内での共有を実現するには、このディレクトリ構造を組織的に管理・配布するメカニズムが必要となる。

npm / GitHub Packagesによる配布

GitHubのプライベートリポジトリでnpmパッケージとしてSkillsを公開し、各プロジェクトで `npm install` することで取得する方法。npm / GitHub Packages詳細で詳しく解説する。

パッケージ構造の例

Skillsをnpmパッケージとして配布する場合、以下のような構造が典型的となる：

@myorg/claude-skills/
├── package.json
├── commands/
│   ├── deploy.md
│   ├── review-pr.md
│   └── generate-tests.md
└── install.js   # .claude/commands/へのシンボリックリンク作成スクリプト

メリット・デメリット

npmによる配布の最大のメリットは、セマンティックバージョニングによる明示的なバージョン管理と、`package.json` による依存関係の記録である。一方、npmレジストリのセットアップ（`.npmrc`設定やGitHub Packages認証）が必要なため、初期コストは高い。

Gitサブモジュールによる共有

Skillsを専用リポジトリに置き、各プロジェクトにGitサブモジュールとして組み込む方法。Gitサブモジュールアプローチ詳細で詳しく解説する。

# 共有Skillsリポジトリをサブモジュールとして追加
git submodule add git@github.com:myorg/claude-skills.git .claude/shared-commands

# プロジェクト側で .claude/commands/ にシンボリックリンクを作成
ln -s ../shared-commands/commands/deploy.md .claude/commands/deploy.md

Gitの標準機能だけで実現できるため、追加ツールが不要という利点がある。ただし、サブモジュールの更新が手動操作を要する点に注意が必要だ。

モノレポ構成による一元管理

社内のすべてのプロジェクトを単一リポジトリで管理するモノレポ構成では、Skillsも同リポジトリ内に配置できる。モノレポでのSkills管理を参照。

セントラルリポジトリからのクローン・コピー

最もシンプルなアプローチは、Skillsを集約したリポジトリを作成し、各開発者がクローンして手動でコピー・シンボリックリンクを張る方法。セントラルリポジトリアプローチで詳細を説明する。

配布自動化とセットアップスクリプト

どの配布方法においても、開発者のオンボーディングを効率化するセットアップスクリプトが重要となる。セットアップ自動化スクリプトでは、シェルスクリプトやMakefileを使った自動化パターンを解説する。

推奨アプローチの選び方

組織の規模やGit運用の成熟度によって最適なアプローチは異なる。小規模チームやGit初心者が多い場合はセントラルリポジトリ + シェルスクリプトが最速で導入できる。中規模以上のチームでバージョン管理を重視するならGitサブモジュールが安定した選択肢だ。すでにnpmエコシステムを活用しているチームにはGitHub Packagesが自然に統合できる。
