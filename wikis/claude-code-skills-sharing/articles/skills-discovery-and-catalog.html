Skills発見性向上のためのカタログとタグ管理

最終更新: 2026-03-31
Claude Code Skillsをチーム全体で効果的に活用するために、タグ付け・カテゴリ分類・カタログ自動生成・検索機能を整備し、Skillsの発見性を大幅に向上させる実践的な手法を解説する。

概要

Skills発見性向上のためのカタログとタグ管理は、組織内でClaude Code Skillsの数が増えるにつれて重要性を増す課題への解決策を提供する。セットアップ自動化によってSkillsのインストールが容易になっても、「どんなSkillsが存在するか」「目的のSkillsをどう見つけるか」が分からなければ活用は進まない。カタログとタグ管理の整備によって、チームメンバーが必要なSkillsを迅速に発見し、重複開発を防ぎ、組織の知識資産を最大化できる。

この記事では、Skillsのメタデータ設計から始まり、タグ付けの標準化、カタログページの自動生成、検索機能の実装まで、発見性向上のための一連の仕組みを構築する方法を解説する。Skills共有方法の概要で紹介した各配布アプローチと組み合わせることで、より強力なSkills管理体制を実現できる。

Skills READMEの標準メタデータフォーマット

各Skillsファイルの先頭に構造化されたメタデータブロックを記述することが、カタログ管理の基盤となる。YAMLフロントマター形式を採用すると、既存のMarkdownパーサーやCIツールとの親和性が高い。

カタログページの自動生成スクリプト

Skillsリポジトリ内のMarkdownファイルを走査し、メタデータを抽出してカタログを自動生成するPythonスクリプトを用意する。セントラルリポジトリアプローチやモノレポ構成いずれにも適用できる。

#!/usr/bin/env python3
# scripts/build-catalog.py - Skillsカタログ自動生成スクリプト
import os
import json
import re
from pathlib import Path
from datetime import datetime

try:
    import yaml
except ImportError:
    import subprocess
    subprocess.check_call(["pip", "install", "pyyaml", "-q"])
    import yaml

SKILLS_DIR = Path("commands")
OUTPUT_DIR = Path("docs/catalog")
CATALOG_JSON = OUTPUT_DIR / "catalog.json"
TAGS_INDEX_JSON = OUTPUT_DIR / "tags-index.json"

def extract_frontmatter(filepath: Path) -> dict:
    """YAMLフロントマターを抽出する"""
    content = filepath.read_text(encoding="utf-8")
    match = re.match(r'^---\n(.*?)\n---', content, re.DOTALL)
    if match:
        try:
            return yaml.safe_load(match.group(1)) or {}
        except yaml.YAMLError:
            pass
    # フロントマターがない場合はファイル名からname推定
    return {"name": filepath.stem, "title": filepath.stem, "tags": []}

def build_catalog() -> dict:
    """全SkillsのカタログJSONを生成する"""
    catalog = {"generated_at": datetime.utcnow().isoformat(), "skills": []}
    tags_index: dict[str, list[str]] = {}

    for skill_file in sorted(SKILLS_DIR.glob("*.md")):
        meta = extract_frontmatter(skill_file)
        meta.setdefault("name", skill_file.stem)
        meta.setdefault("filename", str(skill_file))
        catalog["skills"].append(meta)

        for tag in meta.get("tags", []):
            tags_index.setdefault(tag, []).append(meta["name"])

    catalog["total"] = len(catalog["skills"])
    return catalog, tags_index

def generate_html_catalog(catalog: dict, tags_index: dict) -> str:
    """カタログのHTMLページを生成する"""
    tag_nav = "\n".join(
        f'{tag} ({len(skills)})'
        for tag, skills in sorted(tags_index.items())
    )
    skill_cards = "\n".join(
        f'''

  
/{s['name']}

  
{s.get('title', '')}

  
{s.get('description', '').strip()[:120]}...

  
{''.join(f'{t}' for t in s.get('tags', []))}


'''
        for s in catalog["skills"]
    )
    return f"""
Skills Catalog



Claude Code Skills カタログ


全 {catalog['total']} Skills（最終更新: {catalog['generated_at'][:10]}）


{tag_nav}


{skill_cards}

"""

if __name__ == "__main__":
    OUTPUT_DIR.mkdir(parents=True, exist_ok=True)
    catalog, tags_index = build_catalog()
    CATALOG_JSON.write_text(json.dumps(catalog, ensure_ascii=False, indent=2))
    TAGS_INDEX_JSON.write_text(json.dumps(tags_index, ensure_ascii=False, indent=2))
    html = generate_html_catalog(catalog, tags_index)
    (OUTPUT_DIR / "index.html").write_text(html)
    print(f"Generated catalog: {catalog['total']} skills, {len(tags_index)} tags")

検索機能の実装

カタログに検索機能を追加することで、タグ・タイトル・説明文を横断したSkills検索が可能になる。クライアントサイドのJavaScriptで実装することで、サーバー不要の静的サイトとして運用できる。

検索インデックスの生成

# scripts/build-search-index.py - 検索インデックス生成
import json
from pathlib import Path

def build_search_index(catalog_path: str) -> list[dict]:
    """全文検索用インデックスを生成する"""
    catalog = json.loads(Path(catalog_path).read_text())
    index = []
    for skill in catalog["skills"]:
        index.append({
            "id": skill["name"],
            "title": skill.get("title", ""),
            "description": skill.get("description", ""),
            "tags": skill.get("tags", []),
            "category": skill.get("category", ""),
            # 全フィールドを結合した検索対象文字列
            "searchable": " ".join([
                skill.get("name", ""),
                skill.get("title", ""),
                skill.get("description", ""),
                " ".join(skill.get("tags", [])),
                skill.get("category", ""),
            ]).lower()
        })
    return index

if __name__ == "__main__":
    index = build_search_index("docs/catalog/catalog.json")
    Path("docs/catalog/search-index.json").write_text(
        json.dumps(index, ensure_ascii=False, indent=2)
    )
    print(f"Search index built: {len(index)} entries")

フロントエンド検索UI

// catalog-search.js - カタログ検索フロントエンド
(async () => {
  const resp = await fetch('search-index.json');
  const index = await resp.json();

  const input = document.getElementById('skill-search');
  const results = document.getElementById('search-results');

  input.addEventListener('input', () => {
    const query = input.value.trim().toLowerCase();
    if (!query) {
      results.innerHTML = '';
      return;
    }
    const matched = index.filter(s => s.searchable.includes(query));
    results.innerHTML = matched.length > 0
      ? matched.map(s => `
          

            /${s.id}
            ${s.title}
            

              ${s.tags.map(t => `${t}`).join('')}
            

          
`).join('')
      : `
「${query}」に一致するSkillsが見つかりませんでした。
`;
  });
})();

Skills READMEの標準フォーマット

Skillsの発見性を高めるもう一つの柱が、READMEの標準化だ。npm / GitHub PackagesやGitサブモジュール経由で配布する場合でも、一貫したREADMEフォーマットがあればユーザーが使い方を素早く把握できる。

# /skill-name - Skills名（一行で機能を説明）

## 概要
このSkillsが何をするものか、どんな課題を解決するかを2〜3文で記述する。

## 使い方
```
/skill-name [必須引数] [--オプション値]
```

## 引数・オプション
| 引数 | 必須 | 説明 |
|------|------|------|
| target | Yes | 対象ファイルまたはディレクトリ |
| --focus | No | 重点チェック項目（security/performance/style） |

## 使用例
```
# 基本的な使い方
/skill-name src/components/

# セキュリティに特化したチェック
/skill-name src/ --focus=security
```

## 前提条件
- Claude Code 1.0以降
- プロジェクトにGit管理が必要

## タグ
`code-review` `quality` `security`

- 1.0.0 (2025-11-01): 初回リリース

CI/CDによるカタログ自動更新

Skillsリポジトリへのコミット時に自動的にカタログを再生成し、常に最新状態を維持するGitHub Actionsワークフローを設定する。これにより手動でのカタログ更新作業が不要になり、発見性が常に保たれる。

# .github/workflows/update-catalog.yml
name: Update Skills Catalog
on:
  push:
    branches: [main]
    paths:
      - 'commands/**/*.md'
      - 'scripts/build-catalog.py'
      - 'tags-taxonomy.yml'

jobs:
  build-catalog:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: pip install pyyaml

      - name: Build catalog
        run: |
          python scripts/build-catalog.py
          python scripts/build-search-index.py

      - name: Validate tag taxonomy
        run: python scripts/validate-tags.py

      - name: Commit updated catalog
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add docs/catalog/
          git diff --staged --quiet || git commit -m "chore: update skills catalog [skip ci]"
          git push

      - name: Post summary to Slack
        if: success()
        run: |
          TOTAL=$(python -c "import json; d=json.load(open('docs/catalog/catalog.json')); print(d['total'])")
          echo "Catalog updated: ${TOTAL} skills"

タグ検証スクリプト

未定義のタグや表記揺れを防ぐため、タグ分類体系に基づいてSkillsファイルのタグを検証するスクリプトをCIに組み込む。

#!/usr/bin/env python3
# scripts/validate-tags.py - タグ整合性検証
import json, sys, yaml
from pathlib import Path

taxonomy = yaml.safe_load(Path("tags-taxonomy.yml").read_text())
valid_tags = {tag for cat in taxonomy["categories"].values() for tag in cat["tags"]}

catalog = json.loads(Path("docs/catalog/catalog.json").read_text())
errors = []
for skill in catalog["skills"]:
    for tag in skill.get("tags", []):
        if tag not in valid_tags:
            errors.append(f"[{skill['name']}] 未定義タグ: '{tag}'")

if errors:
    print("タグ検証エラー:")
    for e in errors:
        print(f"  - {e}")
    print(f"\n有効なタグ一覧: {', '.join(sorted(valid_tags))}")
    sys.exit(1)
else:
    print(f"タグ検証OK: {len(catalog['skills'])}個のSkillsが有効なタグを使用")
