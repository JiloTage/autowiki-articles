Verdaccioによるプライベートnpmレジストリの自社運営

最終更新: 2026-03-31
オープンソースのnpmプロキシ/レジストリサーバーVerdaccioを社内に自前でホスティングし、Claude Code Skillsを完全にプライベートな環境で配布する方法。GitHubへの依存を排除しつつ、npmエコシステムの利便性を維持できる。

概要

Verdaccioによるプライベートnpmレジストリの自社運営は、サードパーティサービスに依存せず自社インフラ上にnpmレジストリを構築してClaude Code Skillsを配布するアプローチだ。Skills共有方法の概要で述べたnpmベースの配布手法の一形態であり、GitHub Packagesを使った配布と同じnpmプロトコルを使いながら、ホスティング先を完全に社内に置ける点が最大の差別化要素となる。

Verdaccioはオープンソース（MIT）のNode.js製ソフトウェアで、プロキシ・キャッシュ・プライベートパッケージホスティングの3機能を1つのプロセスで提供する。インターネット疎通のないオンプレミス環境や、外部サービス利用が制限されるエンタープライズ環境での運用に特に適している。

Dockerによるデプロイ

Docker Composeによるセットアップ

最も手軽な運用方法はDockerコンテナを使ったデプロイだ。以下の docker-compose.yml を用意する：

version: "3.8"
services:
  verdaccio:
    image: verdaccio/verdaccio:5
    container_name: verdaccio
    ports:
      - "4873:4873"
    volumes:
      - ./verdaccio/conf:/verdaccio/conf
      - verdaccio-storage:/verdaccio/storage
      - verdaccio-plugins:/verdaccio/plugins
    environment:
      - VERDACCIO_PORT=4873
    restart: unless-stopped

volumes:
  verdaccio-storage:
  verdaccio-plugins:

設定ファイル（config.yaml）

./verdaccio/conf/config.yaml に最低限の設定を記述する：

storage: /verdaccio/storage
auth:
  htpasswd:
    file: /verdaccio/conf/htpasswd
    max_users: 100

uplinks:
  npmjs:
    url: https://registry.npmjs.org/

packages:
  '@myorg/*':
    access: $authenticated
    publish: $authenticated
    unpublish: $authenticated

  '**':
    access: $authenticated
    proxy: npmjs

logs:
  - {type: stdout, format: pretty, level: http}

web:
  enable: true
  title: MyOrg Private Registry

起動と動作確認

# コンテナを起動
docker compose up -d

# ログ確認
docker compose logs -f verdaccio

# 動作確認（レジストリのping）
curl http://localhost:4873/-/ping

認証設定

ユーザーの作成

Verdaccioはデフォルトで htpasswd ベースの認証を使用する。ユーザーの追加はnpm adduser コマンドで行う：

# レジストリに向けてユーザーを作成
npm adduser --registry http://registry.myorg.local:4873

# または npx verdaccio-htpasswd でハッシュを直接生成してファイルに追記
npx verdaccio-htpasswd -b /path/to/htpasswd alice "secretpassword"

トークン認証（CI/CD向け）

CI環境ではインタラクティブなログインができないため、環境変数にトークンを設定する方式を使う。プロジェクトルートの .npmrc：

@myorg:registry=http://registry.myorg.local:4873
//registry.myorg.local:4873/:_authToken=${VERDACCIO_TOKEN}

LDAPプラグインによる社内認証連携

Active DirectoryやOpenLDAPと連携する場合は verdaccio-ldap プラグインを使用する。config.yaml の auth セクションを以下のように変更する：

auth:
  ldap:
    type: ldap
    client_options:
      url: "ldap://ldap.myorg.local"
      adminDn: "cn=verdaccio,ou=serviceaccounts,dc=myorg,dc=local"
      adminPassword: "${LDAP_ADMIN_PASSWORD}"
      searchBase: "ou=users,dc=myorg,dc=local"
      searchFilter: "(uid={{username}})"

アップストリームプロキシ設定

Verdaccioのプロキシ機能を活用すると、@myorg スコープ以外の公開パッケージも社内レジストリ経由で取得できる。一度取得したパッケージはローカルキャッシュに保存されるため、npmjs.orgへの通信が遮断されても既取得パッケージのインストールが継続できる。

packages:
  # 社内専用スコープ: npmjsへのプロキシなし
  '@myorg/*':
    access: $authenticated
    publish: $authenticated

  # 別スコープは別アップリンクへ委譲
  '@company-oss/*':
    access: $authenticated
    proxy: github-packages

  # それ以外は公開npmjs経由
  '**':
    access: $authenticated
    proxy: npmjs

uplinks:
  npmjs:
    url: https://registry.npmjs.org/
    cache: true
    maxage: 10m
  github-packages:
    url: https://npm.pkg.github.com/
    auth:
      type: bearer
      token_env: GITHUB_NPM_TOKEN

GitHub Packagesとの比較

VerdaccioとGitHub Packagesはどちらもプライベートnpmレジストリとして機能するが、運用面での差異が大きい。どちらを選択するかは組織のセキュリティポリシーとインフラ管理コストのトレードオフによる。セットアップ自動化スクリプトを整備することで、どちらの方式でも開発者の初期設定負担を均等に下げられる。

観点 Verdaccio（自社運営） GitHub Packages
ホスティング 自社サーバー / クラウドVMなど GitHub.com（SaaS）
GitHub依存 なし 必須（Token管理）
初期コスト 高（サーバー構築・運用） 低（GitHubアカウントのみ）
インターネット不要 可能（キャッシュ活用） 不可
認証統合 LDAP / htpasswd / プラグイン GitHub SSO / PAT
公開npmプロキシ 組み込み対応 非対応
Web UI シンプルなWeb UI付属 GitHubリポジトリのUI

運用と監視

ヘルスチェック

Docker Composeにヘルスチェックを追加することで、コンテナの自動再起動と監視ツールとの連携が容易になる：

services:
  verdaccio:
    image: verdaccio/verdaccio:5
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:4873/-/ping"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s

ストレージバックアップ

Verdaccioのパッケージデータは /verdaccio/storage に保存される。Dockerボリュームをホスト側ディレクトリにマウントし、定期バックアップを設定する：

# ホストディレクトリへのマウント（docker-compose.yml）
volumes:
  - /data/verdaccio/storage:/verdaccio/storage
  - /data/verdaccio/conf:/verdaccio/conf

# cronによる日次バックアップ例
0 2 * * * tar czf /backup/verdaccio-$(date +\%Y\%m\%d).tar.gz /data/verdaccio/storage

ログ監視

Verdaccioのログを構造化出力に切り替えることで、fluentdやPromtail経由でLoki/Elasticsearchへ転送できる：

logs:
  - {type: stdout, format: json, level: http}
