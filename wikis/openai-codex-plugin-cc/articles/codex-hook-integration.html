Claude Codeフックシステムとプラグイン統合

最終更新: 2026-04-01
Claude Codeのフックシステム（PreToolUse、PostToolUseなど）を活用してcodex-plugin-ccが実現する自動コードレビューゲートの設置方法を解説する。フックの種類・設定・CI/CDパイプラインへの組み込みパターンを包括的に説明する。

概要

Claude Codeフックシステムとプラグイン統合は、Claude Codeが提供するフックメカニズムを通じて、openai/codex-plugin-ccプラグインがClaude Codeのワークフローに深く組み込まれる方法を指す。フックとはClaude Code内部のライフサイクルイベントに対応して外部プロセス（プラグインサーバーを含む）を呼び出す仕組みであり、ツール実行の前後・ファイル保存・セッション管理など複数のタイミングで発火する。

プラグインアーキテクチャにおいてフックは最も強力な統合ポイントの一つである。codex-plugin-ccはこのフックシステムを活用し、開発者が明示的にコマンドを実行しなくても自動的にコードレビューゲートが設置される仕組みを実現している。本記事ではフックの種類・設定方法・CI/CDパイプラインへの組み込みパターンを詳説する。

図1: Claude Codeフックシステムとcodex-plugin-ccサーバーの統合全体図。各フックイベントは対応するプラグインハンドラへルーティングされる。

フックの種類と用途

Claude Codeが提供するフックは大きく「ツール系フック」「ファイル系フック」「セッション系フック」「コミット系フック」の4カテゴリに分類される。プラグインアーキテクチャのplugin.jsonのhooksフィールドに購読するフック名を列挙することで、プラグインサーバーが各イベントを受信できるようになる。

ツール系フック

ツール系フックはClaude Codeがファイル操作・コード実行・検索などのツールを呼び出す前後に発火する最も重要なフックである。

  • PreToolUse: ツール実行前に呼び出される。プラグインはペイロード（ツール名・引数・対象ファイルパスなど）を受信し、allowまたはblockを返してツール実行の可否を制御できる。codex-plugin-ccはここで書き込み対象ファイルの事前バリデーション（禁止パターン検出、テスト未実装チェックなど）を行う。
  • PostToolUse: ツール実行後に呼び出される。実行結果・変更差分・出力内容がペイロードに含まれる。codex-plugin-ccはこのフックで自動コードレビューゲートを実装しており、ファイル書き込みツール実行後に差分をCodexへ送信してレビューを取得する。

ファイル系フック

  • FileSave: ユーザーがファイルを明示的に保存したとき発火する。自動フォーマット・Lintチェック・ドキュメント自動更新などに利用できる。codex-plugin-ccではオプション機能として、保存時に軽量なスタイルチェックをCodex CLIへ送信する。

コミット系フック

  • PreCommit: Gitコミット実行直前に発火する。プラグインがブロックを返すとコミットは中断される。codex-plugin-ccの最重要ゲートであり、コミット対象の全差分をCodexへ送信し、重大な問題（セキュリティ脆弱性・未処理のTODO・壊れたインポートなど）が検出された場合にコミットを阻止する。詳細は対抗的コードレビューを参照。

セッション系フック

  • SessionStart: Claude Codeセッション開始時に発火する。プラグインの初期化・キャッシュウォームアップ・プロジェクト設定の読み込みなどに利用する。codex-plugin-ccはここでCodex APIの疎通確認とプロジェクト固有のレビューポリシーファイル（.codex-review-policy.yaml）の読み込みを行う。
  • SessionEnd: セッション終了時に発火する。ログ送信・一時ファイルクリーンアップ・セッションサマリーの生成などに利用する。

フックの設定方法

codex-plugin-ccでフックを有効化するには、plugin.jsonマニフェストへの宣言とプラグインサーバー側のハンドラ実装の両方が必要である。以下にそれぞれの設定方法を示す。

plugin.jsonでのフック宣言

plugin.jsonのhooks配列に購読するフック名を列挙する。フックごとにフィルター条件（対象ツール名・ファイルパターンなど）を指定できる。 

{
  "id": "openai/codex-plugin-cc",
  "hooks": [
    {
      "event": "PreToolUse",
      "filter": { "tools": ["write_file", "patch_file", "apply_diff"] }
    },
    {
      "event": "PostToolUse",
      "filter": {
        "tools": ["write_file", "patch_file", "apply_diff"],
        "file_patterns": ["**/*.ts", "**/*.py", "**/*.go", "**/*.rs"]
      }
    },
    {
      "event": "PreCommit",
      "timeout_ms": 30000
    },
    {
      "event": "FileSave",
      "filter": { "file_patterns": ["**/*.ts", "**/*.py"] },
      "optional": true
    },
    {
      "event": "SessionStart"
    },
    {
      "event": "SessionEnd"
    }
  ]
}

filterフィールドを使うことで、すべてのツール実行でフックが発火するのではなく、コード変更を伴うツールのみを対象にできる。optional: trueを指定したフックは、タイムアウトや失敗時にClaude Codeのメイン処理をブロックしない。timeout_msはフック応答の最大待ち時間であり、PreCommitのようにCodex APIを呼ぶ場合は余裕を持った値を設定する。

プラグインサーバー側のハンドラ実装

プラグインサーバーはJSON-RPCメソッド hook/{EventName} を実装する。各ハンドラは受け取ったペイロードを処理し、actionフィールドを含むJSON応答を返す。 

// PostToolUse ハンドラ（TypeScript擬似コード）
async function handlePostToolUse(params: PostToolUseParams): Promise<HookResponse> {
  const { tool_name, result, diff, file_path } = params;

  // ファイル変更を伴うツールのみレビュー対象
  if (!diff || diff.length === 0) return { action: "continue" };

  // Codex APIへ差分を送信してレビュー取得
  const review = await codexClient.review({
    diff,
    file_path,
    policy: projectPolicy.postToolUse,
  });

  if (review.severity === "critical") {
    return {
      action: "warn",
      message: `[codex-plugin-cc] 重大な問題を検出:\n${review.summary}`,
    };
  }

  return {
    action: "continue",
    annotation: review.summary,  // 軽微な指摘はインラインアノテーションとして表示
  };
}

レビューポリシーファイル

codex-plugin-ccはプロジェクトルートに置かれる.codex-review-policy.yamlを読み込み、フックごとの動作をカスタマイズできる。 

preToolUse:
  enabled: true
  block_patterns:
    - "eval("
    - "exec("
    - "__import__("
  warn_patterns:
    - "TODO:"
    - "FIXME:"

postToolUse:
  enabled: true
  severity_threshold: warning   # critical / warning / info
  show_inline_annotations: true

preCommit:
  enabled: true
  block_on: critical            # critical のみブロック
  require_clean_review: false   # true にすると指摘ゼロでないとコミット不可
  excluded_paths:
    - "docs/**"
    - "*.md"

自動コードレビューゲートの仕組み

codex-plugin-ccが実装する自動コードレビューゲートは、PostToolUseフックとPreCommitフックの二段階で構成される。このアーキテクチャにより、コード変更直後の軽量チェックとコミット前の厳密な最終チェックを使い分けられる。

図2: PostToolUseで差分単位の軽量レビューを、PreCommitでコミット全体の厳密レビューを実施する二段階ゲート。

PostToolUseゲートの動作

PostToolUseゲートはファイル変更ツールが実行されるたびに発火し、変更差分をCodex APIへ送信する。APIの応答に重大な問題（severity: critical）が含まれる場合は警告をClaude Codeのチャット欄に表示する。問題がない場合や軽微な場合は処理を中断せず、インラインアノテーションとして指摘内容を添付する。このゲートは開発者のフロー状態を尊重しつつリアルタイムフィードバックを提供する設計になっている。

PreCommitゲートの動作

PreCommitゲートはGitコミット実行直前に発火し、コミット対象の全差分をCodex APIへ送信する。PostToolUseと異なり、このゲートは重大な問題が1件でも存在すればコミットをブロックする。ブロック時はClaude Codeのコンソールに問題一覧と修正提案が表示される。対抗的コードレビューの考え方に基づき、セキュリティ脆弱性・壊れたテスト・未解決のTODOなどを積極的に検出する設定になっている。

CI/CDパイプラインへの組み込み

codex-plugin-ccのフックシステムはローカル開発環境での利用を前提としているが、CI/CDパイプラインへ統合することで組織全体のコード品質ゲートとして機能させることができる。

GitHub Actionsへの組み込み

GitHub ActionsでClaude Codeをヘッドレス実行しcodex-plugin-ccを有効化することで、PRマージ前に自動コードレビューゲートを設置できる。デュアルエージェント開発パターンと組み合わせると、AIが生成したコードをCIレベルで自動審査する体制が整う。 

name: Codex Review Gate
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  codex-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # 全差分取得のため

      - name: Install Claude Code
        run: npm install -g @anthropic/claude-code

      - name: Configure codex-plugin-cc
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          claude-code plugin install openai/codex-plugin-cc
          cat > .codex-review-policy.yaml <<'EOF'
          preCommit:
            enabled: true
            block_on: critical
            require_clean_review: true
          EOF

      - name: Run Codex Review Gate (headless)
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          git diff origin/${{ github.base_ref }}...HEAD > /tmp/pr.diff
          claude-code --headless \
            --plugin openai/codex-plugin-cc \
            --hook PreCommit \
            --diff /tmp/pr.diff

gitフック（.git/hooks）との連携

Claude Codeプラグインのフックシステムはgitネイティブのpre-commitフックとも連携できる。install-hooksサブコマンドを実行すると、codex-plugin-ccがプロジェクトのgit hooksディレクトリにシェルスクリプトを自動配置する。 

# codex-plugin-cc が .git/hooks/pre-commit を自動生成
claude-code plugin run openai/codex-plugin-cc install-hooks

# 生成される .git/hooks/pre-commit の内容（抜粋）
#!/bin/sh
claude-code --headless --plugin openai/codex-plugin-cc --hook PreCommit --staged-only
exit $?

--staged-onlyオプションを指定することで、ステージング済みの差分のみを対象にレビューが実行される。これにより作業中の未完成コードがゲートの対象外となり、誤検知を減らせる。

ポリシー・アズ・コード運用

.codex-review-policy.yamlをリポジトリにコミットしてバージョン管理することで、チーム全体で一貫したコードレビューポリシーを適用できる。ポリシーファイルはプロジェクト特性に応じて言語・フレームワーク・セキュリティ要件ごとに設定を変えられる。プラグインアーキテクチャのパーミッションモデルと組み合わせることで、組織のセキュリティポリシーをコードレベルで強制する体制が実現する。

トラブルシューティングとベストプラクティス

タイムアウト調整

Codex APIの呼び出しはネットワーク環境によりレイテンシが変動するため、plugin.jsonのtimeout_msは余裕を持って設定する。PreCommitフックでは30秒以上を推奨する。タイムアウト発生時の動作はon_timeoutフィールドで制御でき、"allow"（タイムアウト時は通過させる）または"block"（タイムアウト時もブロック）を選択できる。

選択的フック有効化

すべてのフックを常に有効にすると開発フローが遅くなる場合がある。PostToolUseはoptional: trueにして遅延なく開発を継続できるようにし、重要なゲートはPreCommitのみに集約するパターンが実用的である。また、filter.file_patternsでテストファイルや生成ファイルを除外することで不要なAPI呼び出しを削減できる。

デバッグ方法

フックの動作確認にはCODEX_PLUGIN_DEBUG=1環境変数を設定する。これによりプラグインサーバーが受信したフックペイロードと送信した応答をコンソールに出力する。フックが期待通りに発火しない場合はuv run awiki session getでセッション状態を確認し、プラグインが正しく登録されているか検証する。
