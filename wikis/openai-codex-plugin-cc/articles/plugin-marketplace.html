Claude Codeプラグインマーケットプレイス

最終更新: 2026-04-01
Claude Codeプラグインマーケットプレイスは、Claude Code上で動作するサードパーティ・公式プラグインを一元的に検索・インストール・管理するための統合配布基盤である。/plugin marketplaceコマンド群を通じてアクセスでき、openai/codex-plugin-ccをはじめとする多数のプラグインがここから配布される。

概要

Claude Codeプラグインマーケットプレイスは、AnthropicがClaude Codeに組み込んだプラグイン配布・管理システムである。開発者はマーケットプレイスを通じて自作プラグインを公開でき、ユーザーはClaude Codeのターミナルから直接プラグインを検索・インストール・更新・削除できる。

マーケットプレイスは、公式プラグイン（Anthropicまたは著名企業が提供）とコミュニティプラグイン（サードパーティ開発者が提供）の2カテゴリに大別される。openai/codex-plugin-ccはOpenAI公式プラグインとして掲載されており、Claude Codeユーザーから特に注目されている。

プラグインの配布はプラグインアーキテクチャの標準仕様に基づいており、マニフェストファイル（plugin.json）による宣言的定義、バージョン管理、署名検証などの仕組みを備えている。

マーケットプレイスコマンド

マーケットプレイスはClaude Codeのスラッシュコマンド体系を通じて操作する。主なコマンドを以下に示す。

検索とインストール

/plugin marketplace search {キーワード}
/plugin marketplace add {owner}/{plugin-name}
/plugin install {plugin-id}
/reload-plugins

/plugin marketplace addはGitHub形式のowner/repoスラッシュ記法でプラグインを指定する。内部的にはマーケットプレイスレジストリを検索し、対応するプラグインマニフェストを取得してローカル環境へのダウンロードと署名検証を実行する。インストール後は/reload-pluginsでClaude Codeにプラグインを認識させる。

openai/codex-plugin-ccのインストールは以下のコマンド列で完了する： 

/plugin marketplace add openai/codex-plugin-cc
/plugin install codex@openai-codex
/reload-plugins
/codex:setup

管理コマンド

/plugin list
/plugin update {plugin-id}
/plugin update --all
/plugin remove {plugin-id}
/plugin info {plugin-id}

/plugin listはインストール済みプラグインの一覧を表示し、各プラグインのバージョン・有効状態・更新可否を確認できる。/plugin infoはプラグインの詳細情報（マニフェスト内容・提供コマンド・依存関係）を表示する。

レジストリと審査プロセス

マーケットプレイスのレジストリはAnthropicが管理する中央レジストリと、コミュニティが維持するパブリックレジストリの2層構造を採用している。

公式レジストリ

公式レジストリに掲載されるためには、Anthropicによるセキュリティ審査・コード品質審査・マニフェスト仕様への準拠確認が必要である。審査を通過したプラグインには「Verified」バッジが付与され、ユーザーが信頼できるプラグインを識別しやすくなっている。

  • セキュリティスキャン: 既知の脆弱性・マルウェアパターン検出
  • マニフェスト検証: プラグインアーキテクチャ仕様への適合
  • コード署名: 公開鍵基盤（PKI）によるプラグイン署名
  • ライセンス確認: OSS・商用ライセンスの明示

コミュニティレジストリ

コミュニティレジストリは審査が簡略化されており、開発者が自由にプラグインを公開できる。ただし、インストール時にはユーザーへ「未審査プラグイン」である旨の警告が表示される。コミュニティレジストリのプラグインは評価数・ダウンロード数・コミュニティレビューによって品質が担保される仕組みとなっている。

アーキテクチャと配布仕様

マーケットプレイスの技術的な基盤はプラグインアーキテクチャの標準仕様と密接に連携している。プラグインのパッケージング・配布・インストールのライフサイクルは以下の通りである。

マニフェスト仕様

各プラグインはplugin.jsonマニフェストを持ち、プラグインID・バージョン・提供コマンド・必要な権限スコープ・依存関係などを宣言する。マーケットプレイスはこのマニフェストを解析し、プラグイン情報ページの生成・互換性チェック・依存関係の自動解決を行う。 

{
  "id": "codex-plugin-cc",
  "version": "1.2.0",
  "name": "Codex Plugin for Claude Code",
  "publisher": "openai",
  "commands": ["/codex:review", "/codex:rescue", ...],
  "permissions": ["network", "process"],
  "dependencies": {}
}

エコシステムにおける役割

プラグインマーケットプレイスはClaude Codeプラグインエコシステム全体の成長を支える中核インフラである。マーケットプレイスの存在により、開発者はClaude Codeのユーザーベースに直接リーチでき、ユーザーは信頼できる単一の窓口からプラグインを入手できる。

主要プラグイン（2026年4月現在）

  • openai/codex-plugin-cc（公式）— CodexとClaude Codeを接続するブリッジプラグイン。4,200スター
  • anthropic/claude-hooks（公式）— ワークフロー自動化フックの拡張セット
  • community/git-ai-commit（コミュニティ）— AIによるコミットメッセージ自動生成
  • community/test-gen（コミュニティ）— コードから自動テスト生成

マーケットプレイス上のプラグイン数は急増しており、AI開発ツール市場における競争激化を背景に、主要プロバイダーが競合他社のエコシステムへの参入を積極化している傾向が見られる。

セキュリティと信頼モデル

マーケットプレイスはプラグインのセキュリティを多層的に担保している。インストール時に権限スコープの確認ダイアログを表示し、ユーザーが意図しない権限付与を防ぐ設計となっている。

  • コード署名検証: インストール時に公開鍵でプラグインの完全性を検証
  • サンドボックス実行: プラグインはClaude Codeのサンドボックス内で動作し、システムへの直接アクセスを制限
  • 権限スコープ: ネットワーク・ファイルシステム・外部プロセス等の権限を宣言制で管理
  • 脆弱性報告: セキュリティ問題の報告窓口とホットフィックス配布の仕組みを提供
