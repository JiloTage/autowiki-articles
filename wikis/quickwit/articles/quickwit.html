Quickwit

最終更新: 2026-04-05
Quickwitは、Rustで実装されたクラウドネイティブな分散型検索・分析エンジンであり、大規模なログ管理や観測性（オブザーバビリティ）ワークフロー向けに設計されたオープンソースプロジェクトです。

概要

Quickwitは、オープンソースのクラウドネイティブ検索・分析エンジンである。Rustで実装されており、S3などのオブジェクトストレージ上に検索インデックスを構築・管理することで、ペタバイト規模のデータに対してもサブ秒以内の検索を実現する。主にログ管理、分散トレーシング、セキュリティ分析（SIEM）などのオブザーバビリティユースケースを対象としている。

Quickwitは従来のElasticsearch/OpenSearchが抱えるコスト課題（ローカルSSD依存、スケーリングの複雑さ）を解決するために設計された。オブジェクトストレージ上の検索を基本アーキテクチャとすることで、ストレージとコンピュートを分離し、大幅なコスト削減と運用簡素化を実現している。

アーキテクチャ

Quickwitのアーキテクチャは、ストレージとコンピュートの明確な分離を特徴とする。

ストレージレイヤー

検索インデックスはすべてAmazon S3、Google Cloud Storage、Azure Blob Storageなどのオブジェクトストレージに保存される。ローカルディスクはキャッシュのみに使用されるため、ノード障害時のデータロストリスクが排除される。インデックスはイミュータブルな「スプリット（Split）」単位で管理され、各スプリットはApache Tantivy上に構築された転置インデックスと列指向ストレージを含む。

Apache Tantivy

Quickwitの検索エンジンコアはApache Tantivyである。TantivyはRustで実装された全文検索エンジンライブラリで、Apache LuceneをインスパイアしたAPIを提供しながら、Rustの安全性とパフォーマンスを活かしている。Quickwitはこの上に分散レイヤー、インデックス管理、クエリプランナーを構築している。

分散アーキテクチャ

Quickwitクラスタは複数のロールを持つノードで構成される:

  • インデクサー（Indexer）: データソースからドキュメントを取得し、スプリットを生成してオブジェクトストレージにアップロードする
  • サーチャー（Searcher）: スプリットをダウンロード（またはキャッシュ参照）してクエリを実行する
  • コントロールプレーン: クラスタのメタデータ管理とスケジューリングを担当する

ノード間の調整にはメタストアが使用され、PostgreSQLまたはファイルシステムが選択できる。

図1: Quickwitのシステムアーキテクチャ。インデクサー・サーチャー・コントロールプレーンが協調し、オブジェクトストレージを共有バックエンドとして使用する。

主な機能

全文検索とフィルタリング

Quickwitは全文検索、ファジーマッチ、フレーズ検索、範囲クエリ、ブール演算などをサポートする。フィールドタイプとして、テキスト・数値・日時・IPアドレス・JSONオブジェクトなどを扱える。Quickwitクエリ言語はElasticsearch Query DSLと互換性があるため、既存のツールチェーンをそのまま活用できる。

集計（アグリゲーション）

ログ分析では集計クエリが不可欠である。Quickwitはヒストグラム、タームアグリゲーション、メトリクス集計（min/max/avg/sum）などElasticsearch互換の集計APIを提供する。

データ取り込み

データ取り込みは複数のソースに対応している:

  • Kafka: ストリーミング取り込みに最適化されたKafkaソース
  • Pulsar: Apache Pulsarからのストリーミング取り込み
  • File / S3: バッチ取り込み向けのファイルソース
  • Ingest API: REST経由のリアルタイム取り込み

ユースケース

Quickwitは特に次のユースケースに強みを持つ:

ログ管理

大量のアプリケーションログ・システムログを低コストで保存・検索する。Elasticsearch互換APIにより、OpenTelemetryやGrafanaなどの既存可観測性ツールとの統合がシームレスである。

分散トレーシング

OpenTelemetryのOTLPプロトコルをネイティブサポートし、分散トレーシングデータを効率的に格納・検索できる。Jaeger互換のUIからQuickwitをバックエンドとして利用することも可能である。

セキュリティ分析（SIEM）

大量のセキュリティイベントログを長期保存しながら、高速な検索・集計でセキュリティインシデントを調査できる。コスト効率の高いオブジェクトストレージを活用することで、ホットデータとコールドデータの階層管理も容易になる。

他ツールとの比較

Quickwitは特にElasticsearchとの比較で言及されることが多い。主な違いは以下の通りである:

  • ストレージコスト: オブジェクトストレージ利用によりElasticsearchの1/10程度のコスト
  • 取り込みレイテンシ: Quickwitは数秒〜数十秒のレイテンシ（Elasticsearchはほぼリアルタイム）
  • クエリレイテンシ: コールドデータはElasticsearchより遅い場合があるが、キャッシュ効果でホットデータは競争力あり
  • スケーリング: コンピュートとストレージの独立スケーリングが可能

エコシステムと統合

Quickwitは多様なツールとの統合を提供する:

  • Grafana: Quickwit専用のGrafanaデータソースプラグインが提供されている
  • OpenTelemetry Collector: OTLPエクスポーターを通じてトレース・ログを直接送信できる
  • Jaeger: Jaeger互換APIにより既存のJaeger UIを流用可能
  • Vector / Fluent Bit: ログ転送エージェントからElasticsearch互換APIで取り込み可能

ライセンスとコミュニティ

Quickwitは当初Apache 2.0ライセンスで公開されていたが、ライセンスモデルはバージョンによって変遷がある。コアエンジンはオープンソースとして維持されており、GitHubでコミュニティが活発に開発を続けている。
