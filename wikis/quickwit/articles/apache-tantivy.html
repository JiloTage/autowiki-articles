Apache Tantivy

最終更新: 2026-04-05
Apache TantivyはRustで実装された高性能な全文検索エンジンライブラリであり、Apache LuceneをインスパイアしたAPIを持ちながら、Rustの安全性とゼロコスト抽象化によって優れた検索パフォーマンスを実現します。Quickwitのコア検索エンジンとして採用されています。

概要

Apache Tantivyは、Rustで実装されたオープンソースの全文検索エンジンライブラリである。JavaベースのApache Luceneからインスパイアを受けて設計されており、転置インデックスを中心とする検索エンジンの基礎機能をRustのエコシステムに提供する。

TantivyはLuceneと同様に、スタンドアロンの検索サーバーではなくライブラリとして提供される。開発者はTantivyをアプリケーションに組み込み、インデックスの構築・管理・検索処理を直接制御できる。QuickwitはこのTantivyを検索コアとして採用し、その上に分散レイヤーとクラウドネイティブな機能を実装している。

プロジェクトは当初「tantivy」として公開されていたが、Apache Software Foundationへの寄贈プロセスを経て「Apache Tantivy」となった。ライセンスはMITからApache 2.0へ変更されている。

転置インデックスとデータ構造

Tantivyの中核はテキスト検索に特化した転置インデックス（Inverted Index）である。転置インデックスは各用語（トークン）からその用語が出現するドキュメントIDのリスト（ポスティングリスト）へのマッピングを保持する。

ポスティングリストと圧縮

TantivyはポスティングリストをBitpacking（ビットパッキング）とVByte圧縮を組み合わせたアルゴリズムで圧縮する。大量のドキュメントIDを効率的にエンコードすることでディスクI/Oを削減し、クエリ実行速度を向上させる。

位置情報（フレーズ検索向けにトークンの出現位置）も格納でき、フレーズクエリや近接クエリに対応する。位置情報の格納はオプションであり、不要な場合は省略することでインデックスサイズを削減できる。

列指向ストレージ（Fast Fields）

Tantivyでは数値・日時・IPアドレスなどの構造化フィールドを「Fast Fields」として列指向形式で格納できる。これはApache Parquetなどの列指向ストレージに類似した仕組みであり、ファセット集計や範囲フィルタリングを高速化する。

図1: Apache Tantivyのインデックス構造。ドキュメントはアナライザーを経てセグメントに書き込まれ、転置インデックス・Fast Fields・ドキュメントストアの3層で管理される。

BM25スコアリング

TantivyはデフォルトのスコアリングモデルとしてBM25（Best Match 25）を採用している。BM25はTF-IDF（Term Frequency - Inverse Document Frequency）を拡張した確率的情報検索モデルであり、現代の全文検索エンジンで広く使われている。

BM25のスコア計算式は以下の要素に基づく:

  • TF（Term Frequency）: ドキュメント内でのクエリ用語の出現頻度（飽和関数付き）
  • IDF（Inverse Document Frequency）: コーパス全体での用語の希少性
  • 文書長正規化: 長いドキュメントへのペナルティ（k1・bパラメータで調整可能）

Tantivyではこれらのパラメータをスキーマ定義時にカスタマイズできる。Apache LuceneのBM25実装と互換性のある計算が行われるため、LuceneベースのシステムからTantivyへの移行時もスコアの傾向が変わりにくい。

セグメント管理とマージポリシー

Tantivyはインデックスを複数のセグメント（Segment）に分割して管理する。セグメントはイミュータブルなインデックス単位であり、書き込みが完了すると変更されない。新しいドキュメントは新しいセグメントに書き込まれる。

マージポリシー

セグメントが増加すると検索時に複数セグメントを走査するオーバーヘッドが増大するため、バックグラウンドで小さなセグメントを大きなセグメントに統合するマージ（Merge）処理が行われる。Tantivyのマージポリシーは設定可能であり、次の2種類が主に使われる:

  • LogMergePolicy: セグメントサイズの対数に基づいてマージ対象を決定するデフォルトポリシー。LuceneのTieredMergePolicyに相当する
  • NoMergePolicy: 自動マージを無効化。Quickwitのように外部でスプリット管理を行う場合に使用される

Quickwitはオブジェクトストレージ上でイミュータブルなスプリット（セグメントの集合）を管理するため、Tantivyのマージポリシーを無効化し、独自のスプリットマージ戦略を実装している。

クエリタイプとファセット検索

Tantivyは多様なクエリタイプをサポートする:

基本クエリ

  • TermQuery: 特定の用語に完全一致するドキュメントを検索
  • PhraseQuery: 複数の用語が連続して出現するドキュメントを検索
  • BooleanQuery: must / should / must_not を組み合わせた複合クエリ
  • RangeQuery: 数値・日時の範囲に基づいた検索
  • FuzzyTermQuery: 編集距離を使ったあいまい検索
  • RegexQuery: 正規表現パターンによる検索

ファセット検索

Tantivyは階層的なファセット（Facet）フィールドをサポートしており、カテゴリ分類された集計検索が可能である。例えば「カテゴリ/サブカテゴリ」のような階層構造を持つデータに対して、カテゴリごとのドキュメント件数を高速に取得できる。Quickwitはこのファセット機能を活用して、ログの属性（サービス名・ログレベルなど）による集計を実装している。

パフォーマンスとベンチマーク

TantivyはRustの特性を活かし、Apache Luceneと比較しても遜色ない、あるいは上回るインデックス構築・検索速度を実現している。

公式ベンチマーク

Tantivyの公式ベンチマーク（tantivy-benchmark）はWikipedia英語版データセット（約500万ドキュメント）を用いて測定されており、以下の特性が確認されている:

  • インデックス構築速度: Luceneと同等〜1.5倍程度の速度。マルチスレッドインデックス構築に対応
  • クエリレイテンシ: 単純なTermQueryで数ミリ秒以下、複雑なBooleanQueryでも数十ミリ秒以内
  • メモリ効率: GCのないRustのメモリモデルにより、GCポーズがなく一貫したレイテンシを維持
  • インデックスサイズ: Luceneとほぼ同等の圧縮効率

特にGCポーズがないことはレイテンシの安定性において大きな優位点であり、リアルタイム検索応用での予測可能なパフォーマンスを可能にする。

TantivyとLuceneの比較

TantivyとLuceneの主な違いを以下に示す:

  • 実装言語: Lucene はJava（JVM）、Tantivy はRust（ネイティブバイナリ）
  • GCの影響: LuceneはJVM GCの影響を受けるがTantivyは受けない
  • エコシステム成熟度: Luceneは20年以上の歴史を持ち、Elasticsearch/OpenSearchなど多くのシステムで実証済み。Tantivyは比較的新しい
  • プラグイン機能: LuceneはCodecやSimilarity等のプラグインシステムが充実。TantivyはRustの型システムで拡張を提供
  • 言語バインディング: TantivyはPython（tantivy-py）、Node.js（tantivy-node）向けバインディングを提供

関連ツールとエコシステム

Tantivyを中心としたエコシステムには以下のツール・ライブラリが存在する:

tantivy-cli

tantivy-cliはTantivyのコマンドラインインターフェースであり、インデックスの作成・ドキュメントの追加・検索の実行をコマンドラインから行える。開発・デバッグ・小規模な検索要件に適している。

言語バインディング

  • tantivy-py: PythonバインディングであるPyO3を使って実装されており、データサイエンスやプロトタイピングでの活用が可能
  • tantivy-node: Node.jsバインディング。JavaScriptアプリケーションへの組み込みに対応

Quickwitとの統合

QuickwitはTantivyを直接依存として使用し、各スプリット（イミュータブルなインデックスチャンク）内部にTantivyインデックスを格納する。Quickwitは以下のTantivy機能を活用している:

  • 転置インデックスによる全文検索
  • Fast Fieldsによる数値・日時の高速フィルタリングと集計
  • ファセット検索によるカテゴリ集計
  • マルチセグメントの並列クエリ実行

列指向ストレージの活用により、Quickwitは大量のログデータに対する集計クエリを効率的に処理できる。
