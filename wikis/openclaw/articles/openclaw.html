OpenClaw

最終更新: 2026-04-08
OpenClaw is a multiplatform, open-source C++ reimplementation of the classic 1997 platformer game Captain Claw by Monolith Productions, designed to run natively on modern operating systems while preserving the original gameplay experience.

Overview

OpenClaw is a fan-made, open-source recreation of the game engine used in Captain Claw (1997), developed by Monolith Productions. The project rewrites the original engine entirely in modern C++, enabling the game to run natively on Windows, macOS, and Linux without relying on emulation or compatibility layers. Unlike playing the original through DOSBox, OpenClaw compiles and executes as a native binary, taking advantage of modern CPU architectures and operating system APIs.

The project is led by developer pjasicek and is hosted on GitHub, where contributions from the community have extended its feature parity with the original game. OpenClaw does not bundle any original game assets; players must supply the CLAW.REZ archive from a legally owned copy of the original game, making it legally and ethically sound as a preservation tool.

History and Background

Captain Claw was released in 1997 and became a beloved cult classic for its swashbuckling pirate cat protagonist, tight platforming mechanics, and memorable level design. However, as operating systems evolved beyond MS-DOS and early Windows, the game became increasingly difficult to run without workarounds. While DOSBox provided one avenue for running the title, native support for modern systems was absent.

The OpenClaw project emerged from a desire to breathe new life into this classic without relying on binary compatibility hacks. Development proceeded on GitHub as an open-source effort, making it possible for community members to contribute bug fixes, feature enhancements, and platform-specific improvements. By 2020, the engine had reached a level of completeness allowing all 14 original levels to be played through.

Technical Architecture

OpenClaw is built on a layered architecture. At the foundation is the platform layer provided by SDL2 (Simple DirectMedia Layer), which abstracts windowing, rendering, input handling, audio playback, and image loading across operating systems. On top of SDL2, the OpenClaw engine implements a game loop, scene graph, entity-component system, and asset manager.

Physics simulation is handled by Box2D, an open-source 2D rigid body simulation library. Captain Claw's platforming behavior — including character movement, collision response, projectile arcs, and enemy interactions — is implemented using Box2D fixtures and contact listeners, providing physically accurate behaviour that closely mirrors the feel of the original game.

Game data (levels, objects, configuration) is loaded from XML files using the TinyXML library, applying a data-driven design that separates game content from engine logic. This makes it easier for modders to alter level layouts and object properties without recompiling the engine.

Asset Loading and CLAW.REZ

Captain Claw stored all its assets — sprites, sounds, music, and level data — inside a proprietary archive format with the extension .REZ. OpenClaw includes a REZ file reader that extracts and loads these assets at runtime, meaning no asset extraction step is needed. The player simply provides the path to CLAW.REZ from a legitimate copy of the game, and OpenClaw handles the rest.

This design upholds game preservation ethics: the engine itself contains no copyrighted content, and the original assets remain under their original license. Only individuals who legally own the original game can play OpenClaw, as the engine cannot function without the official asset archive.

Gameplay Coverage

OpenClaw implements the full original campaign of Captain Claw, including all 14 levels spanning various themed worlds. Enemy AI behaviors, boss encounter scripting, the treasure collection system, power-up mechanics, and the checkpoint save system are all faithfully reproduced. The core gameplay loop — exploration, combat with the cutlass and pistol, and platforming — functions as in the original.

Certain quality-of-life improvements have been added, such as improved frame rate targeting, resolution scaling, and configurable key bindings, while retaining the core experience. Community contributors have also reported and patched various collision edge cases not present in the original to improve reliability.

Community and Development

The project is open-source under the permissive license on GitHub at github.com/pjasicek/OpenClaw. Bug reports, feature discussions, and pull requests are managed through GitHub Issues and the pull request workflow. Community members have contributed ports to additional platforms, performance improvements, and documentation.

OpenClaw is often cited as a model example of game preservation through reimplementation, demonstrating how an open-source engine rewrite can extend the lifespan of a beloved title while respecting intellectual property boundaries.
