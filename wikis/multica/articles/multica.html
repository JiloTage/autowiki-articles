Multica

最終更新: 2026-04-09
MulticaはAIコーディングエージェントをチームメンバーとして管理するオープンソースプラットフォームで、タスク割り当て・進捗管理・スキル蓄積を通じて人間とAIの協働を実現する。

Multica（マルティカ）は、AIコーディングエージェントをチームメンバーとして統合・管理するためのオープンソース・マネージドエージェントプラットフォームである。エージェントデーモンをローカルマシンで動作させ、Claude Code・Codex・OpenClaw・OpenCodeなどのAI CLIとプロジェクト管理ボードを連携させる。GitHub上で約4,000スターを獲得しており（2026年4月時点）、TypeScript（フロントエンド）とGo（バックエンド）で実装されている。

概要と設計思想

Multicaは「あなたの次の10人の採用はヒトではない（Your next 10 hires won't be human）」というビジョンを掲げ、AIエージェントをコード修正・レビュー・デプロイなどのタスクに「同僚のように」割り当てられる環境を提供する。従来のAIアシスタントとの違いは以下の通りである。

  • チームメンバーとしてのエージェント — エージェントはボード上に表示され、コメントを投稿し、ブロッカーを能動的に報告する
  • 自律的なタスク実行 — エンキュー→クレーム→開始→完了/失敗という完全なライフサイクルをWebSocketリアルタイム通知付きで管理する
  • スキルの複利効果 — エージェントが解決した問題は再利用可能スキルとして蓄積され、チーム全体の能力が向上する
  • ベンダー中立 — 複数のAI CLIを統一ダッシュボードで管理し、特定ベンダーに依存しない

システムアーキテクチャ

Multicaは4層のアーキテクチャで構成されている。

フロントエンド（Next.js）

フロントエンドはNext.js 16（App Router）で実装され、apps/web/に配置される。feature-based architectureを採用しており、app/（ルーティング層）・features/（ドメインビジネスロジック）・shared/（横断ユーティリティ）の3層で構成される。状態管理にはZustandを使用し、ドメインごとに1ストア（auth/workspace/issues/inbox）を持つ。WebSocket接続はリアルタイム同期機能としてfeatures/realtime/が担う。

バックエンド（Go）

server/ディレクトリに配置されたGoバックエンドはChi routerを使用する。タスクライフサイクルの管理はinternal/service/task.goが担い、エンキュー→クレーム→開始→完了/失敗の各遷移でWebSocketイベントをブロードキャストする。データベースアクセスはsqlcで自動生成されたGoコードを経由し、PostgreSQLのpgvector拡張も活用する。

データベース（PostgreSQL + pgvector）

PostgreSQL 17にpgvector拡張を加えた構成を採用する。スキーマはすべてマイグレーション管理され、migrations/に配置される。マルチテナント対応のため、全クエリでworkspace_idによるフィルタリングを行う。

エージェントデーモン

ローカルマシン上で動作するエージェントデーモン（multica daemon startで起動）は、PATH上のAI CLI（claude・codex等）を自動検出してRuntimeとして登録する。デフォルト3秒間隔でサーバーをポーリングし、割り当てられたタスクを実行する。最大同時実行数はデフォルト20件で設定可能。

主要機能

エージェントをチームメンバーとして扱う

Issueのアサイニーは多態型（assignee_type + assignee_id）で、人間メンバーとエージェントの両方を受け付ける。エージェントはボード上で紫背景・ロボットアイコンで表示され、コメントの投稿やIssue作成も可能である。マルチワークスペース対応により、チームごとに独立したエージェント・Issue環境を持てる。

スキルシステム

エージェントが解決した問題は再利用可能スキルとして保存される。スキルはデプロイ・マイグレーション・コードレビューといった反復タスクを自動化し、チームの能力を時間とともに複利的に高める。スキルの定義はskills-lock.jsonで管理される。

導入方法

Multica Cloud

最速の導入方法はManaged Cloudサービス（multica.ai）を利用することで、セットアップ不要ですぐに使い始められる。

セルフホスティング

Dockerを使ったセルフホスティングも提供されている。詳細はセルフホスティングガイドを参照。

git clone https://github.com/multica-ai/multica.git
cd multica
cp .env.example .env
docker compose up -d
cd server && go run ./cmd/migrate up && cd ..
make start

CLI（multica コマンド）

multica CLIはローカルマシンとMulticaサーバーを接続するコマンドラインツールである。Homebrew（macOS/Linux）でインストールできる。

brew tap multica-ai/tap
brew install multica
multica login
multica daemon start

技術スタック

Multicaで採用されている主要技術スタックを以下に示す。

  • フロントエンド: TypeScript, Next.js 16 (App Router), Zustand, shadcn/ui
  • バックエンド: Go, Chi router, sqlc, gorilla/websocket
  • データベース: PostgreSQL 17 + pgvector
  • CI/CD: GitHub Actions（Node 22 / Go 1.24）
  • ビルド: Turborepo + pnpm workspaces
  • E2Eテスト: Playwright

他ツールとの比較

Multicaは「LinearのAIエージェント版」と位置づけられる。Claude Codeのような単体エージェントツールと異なり、タスク管理ボード・マルチエージェント対応・スキル蓄積機能を統合した点が特徴である。またAutoGenやLangGraphのようなマルチエージェントフレームワークと比べ、ソフトウェア開発チームのワークフローに特化した実用的な設計となっている。
