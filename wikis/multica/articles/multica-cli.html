multica CLI

最終更新: 2026-04-09
multica CLIはローカルマシンとMulticaサーバーを接続するコマンドラインインターフェースで、認証・エージェントデーモン管理・Issue操作・アップデートなど、Multicaプラットフォームのすべての操作をターミナルから行えるツールである。

multica CLIは、Multicaプラットフォームとローカルマシンを接続するコマンドラインインターフェース（CLI）ツールである。HomebrewによるmacOS・Linuxへのインストールに対応しており、認証・エージェントデーモンの起動・停止・状態確認・Issue管理・自己アップデートといったMulticaのあらゆる操作をターミナル上で完結させることができる。セルフホスティング環境でもソースからビルドして利用可能である。

概要

multica CLIはMulticaプラットフォームのオペレーション窓口として設計されており、以下の役割を担う。

  • 認証管理 — OAuthブラウザフローおよびトークン直接入力の両方に対応し、90日間有効な認証トークンをローカルに保管する
  • デーモン制御 — ローカルのエージェントデーモンの起動・停止・状態確認・ログ閲覧をワンコマンドで行う
  • Issue操作 — ダッシュボードを開かずにCLIからIssueを作成・管理する
  • 自己アップデート — HomebrewまたはバイナリによるCLI自身の更新を自動判別して実行する
図1: multica CLIのコマンド体系。auth・daemon・issue・updateの4グループで構成される。

インストール

multica CLIはHomebrewを使ったインストールが推奨される。macOSおよびLinuxの両方に対応している。

Homebrewによるインストール

Homebrewが利用可能な環境では、以下のコマンドで専用tapを追加してからインストールする。

brew tap multica-ai/tap
brew install multica

インストール後、multica --versionでバージョンを確認できる。

ソースからのビルド

Homebrewが利用できない環境や、最新開発版を使いたい場合はソースからビルドする。Go開発環境とGitが必要である。

git clone https://github.com/multica-ai/multica.git
cd multica
make build
# ビルド成果物 (./bin/multica) をPATHの通った場所にコピー
cp ./bin/multica /usr/local/bin/multica

セルフホスティング環境でCI/CDパイプラインにCLIを組み込む場合も、このビルド手順が利用できる。

認証

multica CLIを初めて使用する前に認証が必要である。取得したトークンはローカルに保管され、90日間有効である。

ブラウザOAuth認証

デスクトップ環境では標準的なOAuthフローを使用する。コマンドを実行するとブラウザが開き、Multicaアカウントでログインする。

multica login

トークン直接入力（ヘッドレス環境）

SSH接続先のサーバーやCI/CD環境のようにブラウザが使えないヘッドレス環境では、--tokenフラグでトークンを直接貼り付けて認証する。

multica login --token

コマンドを実行するとプロンプトが表示され、Multicaダッシュボードで発行したAPIトークンを貼り付ける。

認証状態の管理

認証状態の確認やログアウトには以下のサブコマンドを使用する。

# 認証状態の確認（有効期限・ユーザー情報を表示）
multica auth status

# 保存済みトークンの削除
multica auth logout

デーモン管理コマンド

エージェントデーモンはローカルマシン上でAI CLIを実行するプロセスである。multica daemonサブコマンド群でその起動・停止・監視を行う。

デーモンの起動

デーモンはデフォルトでバックグラウンドで起動する。

# バックグラウンドで起動（推奨）
multica daemon start

# フォアグラウンドで起動（ログをリアルタイム確認したい場合）
multica daemon start --foreground

起動時にPATH上のAI CLI（claude・codex等）が自動検出され、Runtimeとして登録される。

デーモンの停止

multica daemon stop

デーモン状態の確認

稼働中のデーモンのPID・稼働時間・実行中エージェント数・接続ワークスペース数などを確認できる。

# 人間が読みやすいテキスト形式
multica daemon status

# JSON形式（スクリプトや監視ツールからの利用に適する）
multica daemon status --output json

ログの確認

デーモンが出力するログを確認することで、タスク実行状況やエラーを把握できる。

# 直近50行のログを表示
multica daemon logs

# フォローモード（tail -f 相当）でリアルタイム表示
multica daemon logs -f
図2: エージェントデーモンのライフサイクル。daemon start/stop/status/logsコマンドの関係を示す。

Issueコマンド

Multicaのダッシュボードを開かずに、CLIから直接Issueを作成することができる。

Issueの作成

multica issue create

コマンドを実行するとタイトル・説明・アサイニー（人間またはAIエージェント）などの入力を促すプロンプトが表示される。作成されたIssueはMulticaボードにリアルタイムで反映される。AIエージェントにアサインした場合、エージェントデーモンが自動的にタスクを受け取り実行を開始する。

設定・アップデート

CLIの更新

multica updateコマンドはインストール方法（HomebrewまたはバイナリDL）を自動判別し、最新バージョンにアップグレードする。

multica update

Homebrewでインストールされている場合は内部的にbrew upgrade multica相当の処理が実行される。バイナリインストールの場合はGitHubリリースから最新バイナリを取得して置き換える。

接続先サーバーの設定

セルフホスティング環境に接続する場合は、環境変数または設定ファイルで接続先を指定する。デフォルトの接続先はManaged Cloudサービス（multica.ai）である。

# 環境変数で接続先を指定
export MULTICA_API_URL=https://your-self-hosted-instance.example.com
multica login
