エージェントデーモン

最終更新: 2026-04-09
エージェントデーモンはMulticaプラットフォームのローカルランタイムコンポーネントであり、ユーザーのマシン上で動作してAI CLIを自動検出し、サーバーからタスクをポーリングして実行する常駐プロセスである。

エージェントデーモン（英: Agent Daemon）は、Multicaプラットフォームのローカルランタイムコンポーネントである。ユーザーのマシン上でバックグラウンドプロセスとして常駐し、PATH上のAI CLI（claude・codex・opencode等）を自動検出してRuntimeとして登録する。デフォルトで3秒ごとにMulticaサーバーをポーリングし、割り当てられたタスクを受け取ると分離されたワークスペースディレクトリを作成してエージェントCLIを起動し、実行結果をサーバーへストリーミングする。multica CLIのmultica daemon startコマンドで起動される。

概要

Multicaのアーキテクチャにおいて、エージェントデーモンはサーバー（Go バックエンド）とローカルのAI CLI群を橋渡しする中間レイヤーに位置する。ユーザーはサーバーサイドのダッシュボードやCLIからタスクを作成し、デーモンがそれをポーリングで検知して実際のエージェント処理を実行する。

デーモンはステートレスに設計されており、複数マシンで同一ワークスペースに対してデーモンを並列起動することも可能である。各デーモンインスタンスはサーバーへハートビートを送信し、タスクのクレームはサーバー側でアトミックに制御されるため、二重実行は発生しない。

図1: エージェントデーモンのコンポーネント関係図。サーバーからタスクをポーリングし、AI CLIを起動して分離ワークスペースで実行する。

動作の仕組み

デーモンの動作は大きく「起動フェーズ」と「実行ループ」の2段階に分かれる。

起動フェーズ

デーモン起動時（multica daemon start実行時）に以下の処理が順次行われる。

  1. AI CLI自動検出 — システムのPATH環境変数を走査し、claude・codex・opencodeの各バイナリが存在するかチェックする。検出されたCLIはそれぞれ「Runtime」としてサーバーに登録される。
  2. 設定ロード — ~/.multica/config.toml（または環境変数）からポーリング間隔・ハートビート間隔・エージェントタイムアウト・最大同時実行数などの設定を読み込む。
  3. サーバー認証 — multica loginで保存された認証トークンを使ってサーバーへの接続を確立する。
  4. バックグラウンド化 — デフォルトではプロセスをバックグラウンドへデタッチし、~/.multica/daemon.pidにプロセスIDを保存する。--foregroundフラグ指定時はフォアグラウンドで動作する。

ポーリングループ

起動後、デーモンは以下のループを継続的に実行する。

図2: デーモンのポーリングループフロー。タスクが存在する場合のみワークスペースを作成してCLIを起動する。

ワークスペースの分離

タスクを受け付けると、デーモンは~/.multica/workspaces/{task-id}/に専用ディレクトリを作成する。各タスクは完全に独立したワークスペースで実行されるため、並列実行中のタスク間でファイルが競合することはない。タスク完了後、ワークスペースは自動的にクリーンアップされる（設定で無効化可能）。

結果のストリーミング

エージェントCLIの標準出力・標準エラー出力はリアルタイムでサーバーへストリーミングされる。これにより、ダッシュボードからタスクの進捗をリアルタイムで確認できる。タスクライフサイクルの各状態遷移（開始・完了・失敗）もストリーミング経路を通じてサーバーに通知される。

対応エージェントプロバイダー

デーモンは起動時にPATH上の以下のAI CLIを自動検出する。複数のCLIが検出された場合、すべてがRuntimeとして登録される。タスクはサーバー側の設定に応じて適切なRuntimeへルーティングされる。

CLI名 バイナリ 提供元 備考
Claude Code claude Anthropic 最も広く利用されるデフォルトプロバイダー
Codex CLI codex OpenAI OpenAI APIキーが必要
OpenCode opencode コミュニティ オープンソース実装

デーモンはベンダー中立の設計を維持しており、新たなAI CLIが登場した際にも設定の変更なく自動検出が機能する。再利用可能スキルはプロバイダーに依存せず、任意のRuntimeで実行できるよう設計されている。

設定オプション

デーモンの動作は~/.multica/config.tomlで細かく制御できる。主要な設定項目を以下に示す。

設定キー デフォルト値 説明
poll-interval 3s サーバーへのポーリング間隔。短くするほどタスク開始が速くなるが、サーバー負荷が増加する。
heartbeat-interval 15s サーバーへのハートビート送信間隔。デーモンの生存確認に使用される。
agent-timeout 2h 個々のエージェントタスクのタイムアウト時間。この時間を超えたタスクは強制終了される。
max-concurrent-tasks 20 同時実行できるタスクの最大数。マシンのリソースに応じて調整する。

設定ファイル例

以下は~/.multica/config.tomlの設定例である。

[daemon]
poll-interval = "3s"
heartbeat-interval = "15s"
agent-timeout = "2h"
max-concurrent-tasks = 20

[server]
url = "https://app.multica.ai"

[log]
level = "info"
file = "~/.multica/daemon.log"

デーモン管理コマンド

multica CLIはデーモンを管理するための4つのサブコマンドを提供する。

multica daemon start

デーモンをバックグラウンドで起動する。既にデーモンが起動している場合はエラーを返す。

# バックグラウンドで起動（デフォルト）
multica daemon start

# フォアグラウンドで起動（ターミナルに出力を表示）
multica daemon start --foreground

# ポーリング間隔を変更して起動
multica daemon start --poll-interval 1s

multica daemon stop

実行中のデーモンを停止する。進行中のタスクは完了を待ってから終了する（グレースフルシャットダウン）。

multica daemon stop

multica daemon status

デーモンの稼働状態・実行中タスク数・検出されたRuntime一覧などを表示する。

multica daemon status

# 出力例:
# Daemon: running (PID 12345)
# Uptime: 2h 34m
# Active tasks: 3 / 20
# Runtimes: claude, codex

multica daemon logs

デーモンのログを表示する。デフォルトでは最新50行を表示し、-fフラグでリアルタイム追尾ができる。

# 最新50行を表示
multica daemon logs

# リアルタイムで追尾
multica daemon logs -f

# 行数を指定
multica daemon logs -n 100

ログフォーマット

デーモンのログは~/.multica/daemon.logに保存される。ログはJSON Lines形式で記録されており、ログ集約ツールとの統合が容易である。

{"time":"2026-04-09T10:23:45Z","level":"INFO","msg":"daemon started","pid":12345,"runtimes":["claude","codex"]}
{"time":"2026-04-09T10:23:48Z","level":"INFO","msg":"task claimed","task_id":"t_abc123","runtime":"claude"}
{"time":"2026-04-09T10:25:12Z","level":"INFO","msg":"task completed","task_id":"t_abc123","duration":"1m24s"}
