LangChain概要

最終更新: 2026-04-08
LangChainは2022年に創設されたオープンソースフレームワークで、大規模言語モデル（LLM）を活用したアプリケーションおよびAIエージェントの開発を効率化するための包括的なツールチェーンを提供する。Chains・Agents・Memory・Tools・Callbacksなどのコンポーネントを組み合わせることで、複雑なLLMパイプラインを宣言的に構築できる。

LangChainとは

LangChainは、Harrison Chase によって2022年10月にオープンソースで公開されたLLMアプリケーション開発フレームワークである。Python版（langchain）とTypeScript/JavaScript版（langchain-js）の双方が提供されており、急速に普及してAIエージェント開発の標準的な選択肢の一つとなった。

その設計思想は「コンポーザビリティ」にある。プロンプトテンプレート、LLM呼び出し、ツール実行、メモリ管理などの要素をチェーン（Chain）として繋ぎ合わせ、複雑なワークフローを宣言的に記述できる。AIエージェント概論で述べるエージェントパターンの多くがLangChainのコンポーネントとして実装されている。

主要コンポーネント

LangChainの中核を成すコンポーネントは以下の5つに大別される。これらを組み合わせることで、単純な質問応答から複雑なエージェントアーキテクチャまで幅広いシステムを実装できる。

Chains（チェーン）

Chainはプロンプトテンプレート・LLM呼び出し・出力パーサなどの処理ステップを順序立てて接続するパイプライン抽象である。LangChain Expression Language（LCEL）の導入以降は、パイプ演算子（|）を用いた宣言的な記述が主流となっている。シーケンシャルなチェーンから、条件分岐や並列実行を含む複合的なチェーンまで対応する。

Agents（エージェント）

AgentコンポーネントはLLMがどのツールをいつ呼び出すかを自律的に判断するループ機構を提供する。代表的な実装としてReActパターン（思考→行動→観察を繰り返す）とOpenAI Function Callingベースのエージェントがある。ツール利用アーキテクチャで詳述するように、ツールは検索エンジン・コードインタプリタ・外部APIなど多様な形態をとる。

Memory（メモリ）

Memoryコンポーネントは会話履歴や長期的な文脈情報をLLMの入力に組み込む仕組みを提供する。バッファメモリ（全履歴保持）・サマリーメモリ（要約圧縮）・ベクトルストアメモリ（類似度検索）など複数の実装が用意されており、アプリケーションの要件に応じて選択できる。

Tools と Callbacks

Toolsは外部機能（Web検索・計算・データベース照会など）をエージェントが呼び出せる形に標準化したインターフェースである。MCPプロトコルとの統合も進んでおり、ツール定義の相互運用性が高まっている。Callbacksはチェーン・エージェント実行の各フックにロギングやトレース処理を差し込む仕組みで、LangSmithとの連携に活用される。

LangGraph と LangSmith

LangChainエコシステムの拡張として、マルチエージェント向けのLangGraphと観測性ツールのLangSmithが提供されている。

LangGraph

LangGraphは有向グラフ（DAG/サイクルグラフ）としてエージェントの状態遷移を定義するフレームワークである。ノードとエッジで構成されたグラフ上でLLM呼び出しとツール実行を管理し、複数エージェントが協調するマルチエージェントシステムを構築できる。各ノードは状態（State）を入出力とし、条件付きエッジによって動的なルーティングが可能となる。これはOpenClawが採用するエージェントアーキテクチャのアプローチとも概念的に近い。

LangSmith

LangSmithはLLMアプリケーションのトレース・評価・デバッグを支援するSaaSプラットフォームである。実行ログの可視化、プロンプトのバージョン管理、自動評価パイプラインの構築を一元的に行える。エージェントの挙動を詳細に追跡できることから、本番運用における観測性（Observability）の確保に広く利用される。

OpenClawとの比較・使い分け

OpenClawはLLMエージェントをゲームエンジン的なオープンソースランタイムとして実装しているのに対し、LangChainはLLMパイプライン構築のための汎用ライブラリという位置づけである。両者の主要な相違点を以下に整理する。

観点 LangChain OpenClaw
主な用途 汎用LLMアプリ・RAG・チャットボット 自律エージェントの長期実行・タスク自動化
エージェント制御 LangGraphで柔軟に定義 組み込みループ・計画実行エンジン
ツール連携 独自ToolインターフェースおよびMCP対応 MCP標準準拠（MCPプロトコル）
観測性 LangSmith（SaaS） オープンソース中心・組み込みログ
学習コスト 中〜高（多数の抽象レイヤー） 中（統一APIで直感的）

実際の開発では両者を組み合わせることも可能であり、LangChainのツールエコシステムをOpenClawのエージェントランタイムから呼び出す構成も実現できる。RAGパイプラインや既存のLangChain製コンポーネントを活用しつつ、エージェントのオーケストレーション部分にOpenClawを採用するアプローチが有効な場面も多い。

LangChainエコシステムとコミュニティ

LangChainは広大なインテグレーションエコシステムを持ち、主要なLLMプロバイダ（OpenAI・Anthropic・Google・Mistralなど）、ベクトルデータベース（Chroma・Pinecone・pgvector・Weaviateなど）、およびドキュメントローダ（PDF・Web・データベースなど）への公式コネクタが提供されている。

コミュニティは活発で、GitHubスター数はリリースから急速に増加し、LLMフレームワークの中でも最大規模のユーザーベースを形成している。一方で急速な開発に伴うAPI変更の多さや抽象レイヤーの複雑さがしばしば批判の対象となっており、シンプルなユースケースではより軽量なフレームワークが選ばれることもある。

ユースケースと事例

LangChainは以下のような幅広いユースケースで実績がある。

  • RAG（Retrieval-Augmented Generation）: ドキュメントのベクトル化・検索・LLM回答生成を一体化したパイプライン構築。企業内ナレッジベースへの質問応答システムに多く採用される。
  • コード生成エージェント: コードインタプリタツールと組み合わせ、自然言語指示からコード生成・実行・デバッグを反復するエージェントの実装。
  • カスタマーサポートボット: 製品マニュアルやFAQをRAGで参照し、文脈を保持しながら顧客対応を行う会話型エージェント。
  • データ分析エージェント: SQLデータベースやPandasデータフレームへのツールアクセスを通じて、自然言語でデータ分析を実行するエージェント。AIエージェント概論で解説するタスク指向エージェントの典型例である。
  • マルチエージェントワークフロー: LangGraphを用いて、調査・執筆・レビューなど役割の異なる複数エージェントが協調してドキュメントを生成するシステム。
